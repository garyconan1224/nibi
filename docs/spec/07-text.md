
## 模块 7：文字分支

### 7.1 可勾选的分析任务（5 项）

| 任务 | 默认勾 | 说明 |
|---|---|---|
| 摘要 + 要点 + 金句 | ✓ | 一次 LLM 调用同时生成三类 |
| 联想归纳 | ✗ | 4 个方向 |
| 改写 / 润色 | ✗ | 4 种风格 + 可自定义 |
| 翻译 | ✗ | 常见语言下拉 + 自由填 |
| 多文对比 | ✗（动态出现） | 任务内文本 ≥ 2 篇时显示 |

### 7.2 三种输入方式

| 方式 | 处理流程 |
|---|---|
| **网络链接** | 通用 readability 算法抓正文（去导航 / 广告 / 评论）。微信公众号 / 小红书图文等**反爬严格的平台无法保证可用**——抓不到时提示用户"该页面无法解析，请直接粘贴正文" |
| **本地文件** | 解析 .pdf / .docx / .txt / .md |
| **直接粘贴** | 无需额外解析，直接读取（沿用模块 2.5 输入框规格） |

### 7.3 PDF 图文处理（重要技术点）

**目标**：PDF 不只是抽纯文字，还要保留图片信息（图文混合的脚本、报告、产品手册都需要）。

**候选开源方案**（落地阶段评估二选一）：

| 方案 | 特点 | 推荐场景 |
|---|---|---|
| **marker**（VikParuchuri/marker）| PDF → Markdown，保留图片 + 表格，2024 年广受好评 | 通用首选 |
| **docling**（IBM 开源）| 结构化文档解析，表格识别强 | 偏技术报告 |
| **unstructured** | 通用文档解析，可单独 OCR 图片 | 备选 |
| PyMuPDF (fitz) | 底层库，自行组装 | 兜底 |

**处理流程**：
1. PDF → Markdown（用 marker / docling）
2. 提取的图片**位置保留**（占位符或 base64 嵌入 Markdown）
3. 图片**默认不分析**（作为内嵌资源保留）
4. 用户可在 Preflight 抽屉中开启「PDF 内图片走图片分支分析」选项——此时图片**作为独立素材入库**到当前任务

### 7.4 超长文本检测 & 分段

```
检测文本字数
  ├─ 未超 LLM 上下文 → 直接分析
  └─ 超出 → 按段落 / 章节自动切分
            → 各段独立处理
            → 结果合并 + 二次精炼
```

### 7.5 四个并行任务

#### 任务 1：摘要 / 要点 / 金句

- 一次 LLM 调用同时生成三类
- **摘要长度**：50 字 / 100 字 / 200 字（默认 100 字）
- 要点和金句必须带**可精确跳转的原文位置标注**。
  - 金句：`quote_text` 必须是原文中的精确片段；保存结果前用原文全文做 substring 校验，得到 `char_start` / `char_end` 与段落序号。
  - 要点：生成文案可为归纳句，但必须额外给出 `source_excerpt`（原文精确片段）作为锚点；点击要点时跳到 `source_excerpt`，而不是跳到生成文案的近似位置。
  - UI 中所有可点击的要点 / 金句都必须跳到左侧原文对应位置并短暂高亮；禁止用“段落开头”“全文搜索第一个相似句”等近似锚点冒充精确定位。

#### 任务 2：联想归纳

**4 个固定方向**（用户多选）：
- 深度解读
- 观点提炼
- 趋势判断
- 行动建议

输出：观点 / 趋势 / 建议（比摘要更深层）

#### 任务 3：改写 / 润色 / 翻译

**改写风格**（4 个预设 + 自定义）：
- 正式
- 口语
- 简洁
- 丰富
- **自定义**：用户输入风格指令（如"小红书种草风"）

**翻译目标语言**：
- 下拉清单：中 / 英 / 日 / 韩 / 西 / 法 / 德 / 俄 / 葡
- 「其他」选项 → 用户自由填语言名

输出：**原文与改写/翻译结果并排对照展示**（详情页左原文右结果）

#### 任务 4：多文对比

跟模块 6 多图对比逻辑一致：
- 触发：任务内文本素材 ≥ 2 篇时，preflight 自动出现"多文对比"勾选项
- 入口：「素材」子标签 → 多选文本 → 顶部"对比"按钮
- 对比维度：观点异同 / 立场倾向 / 信息完整性 / 时间线梳理
- 输出：表格 / 报告（用户选）

### 7.6 文字结果交互界面（沿用设计稿）

**布局**（grid: 1fr 1fr 或 1fr 400px）：

```
┌──────────────────────────┬──────────────────────┐
│ 顶部：标题 + 字数 + 阅读时长│ ▼ 摘要               │
├──────────────────────────┼──────────────────────┤
│                          │ ▼ 要点列表           │
│                          │   (含原文位置跳转)   │
│   原文完整展示           ├──────────────────────┤
│   （Markdown 渲染）      │ ▼ 金句               │
│   （直接粘贴的文本也展示）│                      │
│                          ├──────────────────────┤
│                          │ ▼ 联想归纳           │
│                          ├──────────────────────┤
│                          │ ▼ 改写 / 翻译        │
│                          │   (并排对照模式)     │
└──────────────────────────┴──────────────────────┘
```

**改写/翻译模式特殊布局**：在「改写 / 翻译」tab 内做**逐段对照**，每行左侧是原文段落，右侧是当前选择的改写或译文。不是整篇 vs 整篇，也不是常驻三栏挤压阅读区。用户可逐段复制，切换改写风格或目标语言时保持同一段落锚点。

逐段对照的数据要求：
- 结果产物需保存原文分段数组（段落序号 + 原文文本 + 原文 `char_start` / `char_end`）。
- 改写 / 翻译结果优先按同一段落序号返回；若 LLM 输出段数不一致，后端需在落盘前做段落对齐或标记该段缺失，前端不得把整篇结果粗暴塞到单个段落。
- 点击任一对照行的原文侧，同样应滚动并高亮左侧原文对应段落。

### 7.7 异常处理

| 异常 | 处理 |
|---|---|
| 链接抓取失败（反爬 / 404） | 提示"该页面无法解析，请直接粘贴正文" |
| PDF 解析失败 | 退化到 PyMuPDF 纯文字提取 |
| 文本超长 | 自动分段处理后合并 |
| 翻译语言不支持 | LLM 兜底尝试 |

### 7.8 导出节点（绿色节点）

- 原文（.txt / .md）
- 摘要 + 要点 + 金句（.md）
- 联想归纳（.md）
- 改写 / 翻译（.docx）
- 多文对比报告（.md / .json）
