### 8.2 「导出工作包」功能——**代码保留 / UI 隐藏**

经过场景分析判断，导出 zip 工作包**没有强应用场景**：
- 本地归档 → Nibi 已是本地软件
- 分享给朋友 → 朋友没装 Nibi 时 prompts.json 用不起来
- 跨工具流转 → 素材详情页"一键复制"已够用

**结论（修订版）**：
- **现有代码保留**（Phase 1I 已实现的前后端 zip 导出逻辑不删，作为可重启功能）
- **UI 入口隐藏**：
  - Taskboard 不显示「导出」子标签
  - 任务中心 / 顶部都没有"导出工作包"按钮
  - 用户表面上看不到这个能力
- **未来重启条件**：如收到用户反馈"我需要打包带走"，再恢复 UI 入口（成本极低）
- **保留**：单素材级别的导出能力（每个素材详情页可下载字幕 .srt、文案 .md、提示词 .json、原文件等），已在模块 4-7 各自落 spec
- 任务中心子标签调整为：素材 / 队列 / 标签库 / AI 对话（**4 个**，不再是 5 个；模块 1.8 同步更新）

### 8.3 任务总览页 = Taskboard（同一个东西）

不再单独设计"任务总览页"。用户点进任务直接落在 Taskboard，默认子标签 = 「素材」。Taskboard 顶部已包含：
- 任务基本信息（任务名 / eyebrow）
- 5 字段背景条 + 编辑按钮
- 添加素材按钮
- 跨工作空间检索栏（WorkspaceSearchBar）
- 4 个子标签（素材/队列/标签库/AI 对话）

### 8.4 全流程进度可视

#### 8.4.1 步骤级进度

每个 pipeline 步骤实时显示：
```
✓ 下载（已完成 5 秒）
✓ 转码（已完成 12 秒）
⟳ 截帧（进行中 67%，预计还需 8 秒）
○ 视觉分析（等待中）
○ LLM 总结（等待中）
○ 入库（等待中）
```

状态：已完成 ✓ / 进行中 ⟳ / 等待中 ○ / 失败 ✗
颜色：等待中 灰 / 进行中 蓝（带动画）/ 已完成 绿 / 失败 红

#### 8.4.2 进度展示位置

- **单素材进度**：素材卡片角上的小进度条 + 状态点
- **任务整体队列**：Taskboard 的「队列」子标签（不做全局批量面板）
- **实时推送**：SSE + WebSocket（已实现）

#### 8.4.3 性能自适应

启动时检测：
- CPU 核心数
- 内存大小
- GPU 性能（如有）

自动推荐合理并行数上限，用户可在「设置 → 部署监控」里手动调整。

### 8.6 异常处理汇总表

各模块的异常已分别落 spec，此处汇总：

| 异常类别 | 处理 | 所属模块 |
|---|---|---|
| 链接下载失败（版权/地区/网络） | 明确提示，支持重试 | 模块 2 |
| 平台不支持 | 提示用户，尝试通用网页抓取 | 模块 2 |
| 文件损坏 / 不完整 | 提示重传 | 模块 2 |
| 格式不支持 | 后台自动转码，用户无感知 | 模块 2 |
| 视觉模型 API 失败 | 重试 3 次，超限标注跳过该帧 | 模块 4 / 6 |
| Whisper 转空（无人声且未勾音乐分析） | 弹窗提示切换音乐分析 | 模块 5 |
| pyannote 说话人分离失败 | 退化为单一说话人 | 模块 5 |
| 音乐特征检测失败 | 回退 90 秒固定切片 | 模块 5 |
| OCR 引擎不可用 | 回退到视觉模型 OCR | 模块 6 |
| EXIF 不存在 | 显示"无 EXIF 信息" | 模块 6 |
| 链接抓取失败（反爬） | 提示"请直接粘贴正文" | 模块 7 |
| PDF 解析失败 | 退化到 PyMuPDF 纯文字提取 | 模块 7 |
| 文本超出 LLM 上下文 | 自动分段处理后合并 | 模块 4 / 7 |
| 批量任务单个失败 | 标红，单独重试，不影响其他 | 模块 2 |
| 模型未配置 | 提示用户去设置添加模型 | 模块 3 |
