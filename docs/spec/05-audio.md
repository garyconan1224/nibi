
## 模块 5：音频分支

### 5.1 可勾选的分析任务（5 项）

| 任务 | 默认勾 | 说明 |
|---|---|---|
| 人声转写 + 内容总结 | ✓ | Whisper + LLM |
| 说话人音色区分 | ✓ | 声纹聚类（pyannote.audio） |
| 生成字幕文件 | ✓ | .srt / .txt |
| **是否进行音乐分析** | ✗ | 用户主动勾选，无论有无人声 |
| 生成音乐提示词 | ✗（依赖音乐分析勾选） | Suno/Udio 通用格式 |

### 5.2 VAD 人声检测分流

```
音频上传 → VAD 人声活动检测
  ├─ 有人声 → 人声处理路径（5.3）
  └─ 无人声 / 纯音乐
       ├─ 用户已勾音乐分析 → 直接走音乐分析路径（5.4）
       └─ 用户未勾音乐分析 → 弹窗提示"未检测到人声，是否切换为音乐分析模式？"
            └─ 用户确认 → 启用音乐分析 + 持久化到该素材 preflight
```

### 5.3 人声处理路径

**Whisper 参数**（来自 Preflight 抽屉 / 设置页默认）：
- 语言：自动检测 / 中文 / 英文 / 其他
- 说话人分离：开 / 关（会议、采访场景推荐开）

**说话人识别**（技术选型）：
- 用 **pyannote.audio**（开源主流方案）做声纹聚类
- 输出"检测到 N 个说话人"
- 结合任务背景信息中的「参与人员」字段尝试自动匹配人名
- 弹窗让用户**确认 / 修正**说话人标签（每个说话人显示一段示例音频）

**转写输出格式**：
```
[00:00:05][张总] 大家好，今天我们讨论 Q3 战略...
[00:00:18][李总] 我觉得首先要明确目标客户...
```

**文本清洗**：
- 去语气词 / 合并短句 / 修正识别错误
- 注入任务背景的「专有名词」做修正（与模块 4 字幕清洗机制一致）

**字幕导出**：
- `.srt`：带时间戳和说话人标注
- `.txt`：纯文本（可选是否带说话人）

**总结输出（按内容类型套模板）**：

从任务级背景信息「内容类型」自动选择模板，**不让用户在此重复选**：

| 内容类型 | 输出字段 |
|---|---|
| 播客 | 摘要 / 要点 / 金句 / 话题亮点 / 嘉宾观点 |
| **会议录音** | 摘要 / 要点 / 金句 + **待办事项 / 决策记录 / 发言人小结**（会议特有） |
| 课程讲解 | 摘要 / 要点 / 金句 / 知识点 / 重点概念 |
| 演讲 | 摘要 / 要点 / 金句 / 核心论点 |
| 采访 | 摘要 / 要点 / 金句 / 受访者观点 / 问答摘要 |

超长文本自动分段处理后合并。

### 5.4 音乐分析路径

**触发条件**：
- 用户主动勾选了「音乐分析」
- 或视频路径 2 + 用户勾了音乐分析（详见 5.5）
- 或 VAD 检测无人声 + 用户在弹窗确认切换

**片段切分（技术选型）**：
- 采用**音频特征自动检测**（BPM 变化、能量变化）做切分
- 切分失败 / 检测异常时回退「固定时长 90 秒一段」
- 切分粒度：每段约 1-3 分钟，避免过细

**每段音乐 6 维度分析**：
- 风格（流行/爵士/电子/古典 等）
- 情绪（欢快/悲伤/激昂/舒缓 等）
- BPM（每分钟节拍数）
- 主要乐器
- 调性（C 大调 / A 小调 等）
- 整体氛围（温暖/冷峻/神秘 等）

**LLM 生成三类输出**：
1. 相似曲风参考（推荐风格方向）
2. 适合使用场景（广告 / Vlog 配乐 / 电影配乐 等）
3. **AI 音乐生成提示词**（一份通用格式，兼容 Suno 与 Udio——两者格式差异不大，不分开输出）

**输出 JSON 示例**：
```json
{
  "segments": [
    {
      "start": "00:00:00", "end": "00:01:30",
      "style": "indie folk", "mood": "nostalgic, warm",
      "bpm": 92, "instruments": ["acoustic guitar", "soft vocals"],
      "key": "G major", "atmosphere": "intimate, contemplative",
      "music_prompt": "indie folk, acoustic guitar, soft male vocals, 92 bpm, ..."
    }
  ],
  "overall_style": "modern indie folk with cinematic touches",
  "similar_references": ["Bon Iver", "Sufjan Stevens"]
}
```

### 5.5 与视频路径 2（音视频合并）联动

**重要变更**——不强制：

- 视频路径 2 默认**只用画面描述 + 字幕合并**，不强制跑音乐分析
- 如果用户在视频素材的 preflight 中**勾选了音乐分析**，路径 2 才合并音乐数据生成"音画提示词"
- 没勾就照原始勾选走（尊重用户选择，不强加未勾任务）

### 5.6 音频结果交互界面（沿用设计稿）

布局：
```
┌────────────────────────────────────────┬──────────────┐
│ 顶部：标题 + 时长 + 导出按钮            │ 右侧总结面板  │
├────────────────────────────────────────┤              │
│ 波形图（Waveform，播放进度跟随）        │ abstract     │
├────────────────────────────────────────┤ keyPoints    │
│ 转写区（每行：时间戳 + 说话人色块 + 文本）│ quotes       │
│                                        │ todos（会议） │
├────────────────────────────────────────┤ decisions    │
│ 音乐片段卡列表（每张含 6 维度 + 复制按钮）│ speakerSummary│
└────────────────────────────────────────┴──────────────┘
```

**核心交互**：
- 点击波形某处 / 转写行 → 跳转播放位置
- 转写当前行自动高亮
- 音乐片段卡的"复制 Suno 提示词"按钮
- 说话人色块用任务背景的人名（已匹配的）

**后续可能改的方向**（先做现状，根据使用反馈再调）：
- 音乐片段卡改成第二条时间轴对齐字幕
- 增加波形上"说话人色带"可视化

### 5.7 导出节点（绿色节点）

- 原音频文件（.mp3 / .wav）
- 提取后的字幕文件（.srt / .txt，带说话人）
- 最终总结（.md / .json）
- 音乐分析结果（.json，含 music_prompt）

### 5.8 异常处理

| 异常 | 处理 |
|---|---|
| Whisper 转空（确实无人声且未勾音乐分析） | 弹窗提示切换音乐分析 |
| pyannote 聚类失败 | 退化为单一说话人模式，记录失败原因 |
| 音乐特征检测失败 | 回退固定 90 秒切片 |
| 文本超长 | 自动分段处理后合并 |
