
## 模块 2：输入层

### 2.1 入口与整体交互

**「添加素材」模态——4 步合一**（采用设计稿 UI 形态，新手不用跳多个页面）：

```
① 素材类型 ──→ 视频 / 音频 / 图片 / 文字
② 输入源   ──→ URL 输入框 + 上传按钮 + 拖拽区
③ 勾选分析任务 ──→ 按所选类型显示对应任务项
④ 背景信息（折叠可选）──→ 5 字段，提交时注入到所有 AI 调用
```

**交互优化（重要）**——「混合方案」：
- 用户**粘贴 URL 后**，系统自动识别平台和内容类型，**自动回填 ① 素材类型**（用户不用先选）
- 用户**拖入本地文件后**，根据 magic bytes 识别真实格式，**自动回填 ① 素材类型**
- 用户也可以**先点 ① 选类型再粘贴**（两种入口都通）

**支持的输入**：
- **网络链接**：B 站 / 小红书 / 抖音 / 快手 / YouTube / 通用网页文章
- **本地文件**：视频 / 音频 / 图片 / 文字（pdf/docx/txt/md）
- **直接粘贴文本**：仅文字类型

### 2.2 链接处理流程

```
用户粘贴 URL
  ↓
URL 格式验证 + 平台识别（域名匹配）
  ↓
内容类型检测
  ├─ 单一类型（纯视频/纯文章）→ 自动回填素材类型，继续
  └─ 混合类型（如 B 站视频可下视频/音频/同时）→ 弹窗让用户选下载哪些
  ↓
（视频）画质：默认走「设置」页的全局默认（首次默认最高）
  ↓
yt-dlp 执行下载 + 实时进度
  ↓
完成 → 原文件可单独导出
```

**画质策略**（Q2 决议）：
- **默认**：最高画质
- **可配置**：在「设置」页有"默认下载画质"全局设置（最高 / 1080p / 720p / 仅音频）
- 添加素材模态里**不显示画质选项**，简化新手交互；高级用户去设置里调

**混合类型弹窗**（Q3 决议）：
- B 站等支持视频/音频分离下载的平台，**保留这个弹窗**
- 弹窗选项：下载完整视频 / 仅下载音频 / 同时下载两者（各生成一个素材）

**异常处理**：
- 下载失败（版权 / 地区 / 网络）→ 明确提示原因，支持重试
- 平台不支持 → 尝试通用网页抓取，仍失败则提示

### 2.3 本地文件处理流程

```
用户拖拽或选择文件
  ↓
文件大小及完整性校验（本地版无大小限制；损坏文件 → 提示重传）
  ↓
格式识别（magic bytes 判断真实格式，不信赖后缀）
  ↓
格式判断
  ├─ 已支持格式 → 直接进入下一步
  └─ 不支持格式 → 后台自动转码，用户无感知（显示进度）
  ↓
读取完成 → 原文件可单独导出
```

### 2.4 批量上传（Q5 决议）

- **URL 一次只能粘贴一个**（强制用户一个个加，避免误操作）
- **本地文件支持多选 / 拖拽多个**——一次最多 N 个（N 建议 20，可在设置调）
- 多文件上传后，每个文件**独立成为一个素材**，分别进队列
- 多个素材的任务勾选**共用同一次模态的 ③ 勾选项**（用户填一次，应用到所有）

### 2.5 文本素材的输入框（Q6 决议）

- 文本输入框**保持单行 / 小尺寸**外观（不变形为大 textarea）
- 用户粘贴大段文字时：
  - 输入框内只显示**首行可见部分**（其余字符隐藏，但内部完整保留）
  - 右侧显示「已粘贴 N 字」的提示
  - 不允许用户在框内编辑，只能"清空"
- 分析完成后，进文字详情页才看到完整文本

### 2.6 任务勾选默认（Q7 决议）

**智能默认**：根据素材类型，预先勾选最常用的分析项：

| 素材类型 | 默认勾选项 |
|---|---|
| 视频 | 画面提示词生成 ✓ / 视频文案总结 ✓ / 字幕导出 ✓ / 音乐分析 ✗ |
| 音频 | 人声转写+总结 ✓ / 字幕导出 ✓ / 音乐分析 ✗（除非 VAD 检测无人声自动切换） |
| 图片 | 内容识别 ✓ / OCR ✓ / 提示词生成 ✓ / 联想总结 ✗ |
| 文字 | 摘要+要点+金句 ✓ / 改写 ✗ / 翻译 ✗ / 多文对比 ✗ |

**用户可在「设置」页自定义"我的默认勾选偏好"**——下次添加素材时按用户设的默认走，不按系统智能默认。

### 2.7 队列管理（Q4 决议）

- **不做独立的"批量队列面板"一级模块**
- 任务队列状态进**任务中心的「队列」子标签**
- 该子标签提供：
  - 整体进度（N 个素材在跑 / M 个排队 / K 个失败）
  - 单素材进度条 + 当前步骤名
  - 单素材操作：取消 / 失败重试
  - **不做拖拽排序优先级**（对新手价值不大）

### 2.8 共用后续逻辑

```
两条路径汇合
  ↓
统一内容类型路由（视频/音频/图片/文字 → 四大分支）
  ↓
进入对应分支的处理流程（详见模块 4-7）
```
