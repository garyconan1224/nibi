## 1. 【入口】「＋新建」直接弹出添加素材弹框（图1→图2）

- 诉求：点侧栏「＋新建」**直接打开添加素材弹框**（输入链接），而不是只跳首页。
- 架构问题：弹框现在是 `Composer`（首页）局部状态，从任意页点「新建」无法直接开。需要让弹框可全局触发。**择一（codex 判断，优先 a）**：
  - **(a) 弹框开关提到全局**：用一个轻量全局 store（zustand，与项目现有 store 风格一致）或 context 管理 `addMaterialOpen`；`AddMaterialModal` 提升到 `AppShell` 级渲染一次；「＋新建」与首页「添加素材」都 set 这个开关。
  - **(b) 最小改**：「＋新建」`navigate('/?new=1')`，`Composer` 读 query 自动 `setUploadOpen(true)` 并清掉 query。
- 验收：任意页点「＋新建」→ 弹出图2 弹框；首页原有「添加素材」入口仍可用、行为一致。
- ⚠️ 不改弹框内部提交逻辑/契约，只改「谁来开它、在哪渲染」。

## 2. 【首页简化】移除内联「归入合集」行

- 诉求：首页更简单，**去掉「归入合集 / 不归入任何合集 / 选择合集」整行**（`Composer.tsx:239-268`）。合集选择只在弹框里做（弹框已有「选择合集 / ＋新建合集」）。
- 首页 Composer 保留：粘贴输入框 + 上传 + 「添加素材」按钮。
- 弹框打开时合集默认仍为「不属于任何合集」（见 §3），不再从首页带入预选。
- 验收：首页只剩输入框/上传/添加素材；不再有合集行；功能不回归。

## 3. 【合集逻辑】默认不属于任何合集，下拉选才归属

- 诉求重申：**默认不选合集 = 不属于任何合集（收纳箱/单独）**；只有在弹框「②合集归属」里**下拉「选择合集」**才归到某合集。
- 现状基本已实现（默认 `workspaceSel` 空 → 显示「收纳箱（单独素材）」）。Codex 校验 + 收口：
  - 默认 `workspaceIds` 为空、文案为「收纳箱（单独素材）」或「不属于任何合集」（与首页措辞统一，二选一即可、保持一致）。
  - 「选择合集」下拉选中后才归属并显示合集名 + 类型；「＋新建合集」保留。
  - 未选合集时「笔记/复刻」均可选（动作不被锁）。
- ⚠️ 若「无合集=单独」涉及后端提交语义（空 workspaceIds 后端如何落库）需改契约 → **停下标注**，仅做展示层。

## 4. 【截帧 + 模型常驻 + 取画面 UI 精简】（本轮重点，部分推翻上一轮折叠）

- 诉求：上一轮把「取画面 / 视觉模型」折叠进了「高级设置」。用户说这俩**经常用、不该隐藏、要常驻可见**；且「取画面」UI 要**更精简好看、不占空间**。
- 改造：
  - 把「**视觉模型** + **取画面（截帧参数）**」从「高级设置」accordion 里**提出来，常驻显示**在 ④ 笔记设置/复刻设置区（仅在该动作真正需要时显示——视频笔记 / 逐帧复刻；音频/图文不显示）。
  - 「高级设置」折叠里**只保留低频项**：区分发言人、补充说明（及其它不常用项）。
  - **取画面 UI 重做（更紧凑）**：当前 `智能/手动` 两张大卡 + 间隔输入 + 预估帧数占了不少竖向空间（见图4）。改成**一行式紧凑控件**，例如：
    - 一个「智能/手动」小分段切换（segmented），后面紧跟「每 [N] 秒」步进 + 「约 N 张」预估，单行排布；
    - 「按时长自动」作为智能模式下的副标说明，弱化；
    - 复用 Nibi token、对照设计稿 `.am-*` 取画面块，但目标是比设计更精简、占用更小。
  - 让 codex（GPT-5.5）自己把这块 UI 做得克制、好看、信息密度高、不占竖向空间。
- 验收：视频笔记/复刻时「视觉模型 + 取画面」**首屏直接可见**、无需展开；取画面控件紧凑成 1~2 行；切「智能/手动」「间隔」实时更新「约 N 张」预估；音频/图文笔记不显示这些；提交参数（embed_frames / 取画面间隔 / 视觉模型 / replica_kind 等）**与改前一致**。
