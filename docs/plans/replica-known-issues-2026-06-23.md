## 问题 1：处理页（任务中心）对复刻任务套用了笔记文案/步骤

**现象**：选「复刻」跑完，处理页 stepper 显示「排队 → 下载 → **音频转录** → 画面分析 → **生成笔记** → 完成」，完成提示「**笔记已生成完毕**，正在打开…」。但实跑日志是 `steps=['download', 'analyze']`（转写+笔记已正确裁掉）——**UI 步骤与真实执行不一致，且文案是笔记向**。

**期望**：步骤随任务类型/二级动态变化（对齐手测 #19）。复刻提示词应是「下载 → 截帧 → 画面提示词 → 完成」（在线视频前面加排队；本地去掉下载）。

**根因方向**：`ProcessingPage` 的步骤矩阵 + 完成文案按 note 写死，未按 `intent=replica` / `replica_kind` 分流。**与手测 #19（任务中动态步骤）同源**——#19 当时只覆盖笔记，复刻是新增分支未覆盖。

**证据**：截图（stepper 含「音频转录/生成笔记」）；实时日志 `steps=['download','analyze']` + 「笔记已生成完毕」。

## 问题 2：复刻结果页提示词含未填充的模板占位符

**现象**：复刻页右侧「当前提示词」显示
`Bilibili首页界面，…，{tags.style}, {tags.lighting}, {tags.composition}, --ar 16:9 --style raw`
——`{tags.style}` `{tags.lighting}` `{tags.composition}` 是**原始模板变量，没被填值**（页面「内容标签」显示「暂无标签」，tags 没生成）。

**期望**：要么复刻流程真生成 tags 并填入；要么提示词渲染对缺失 tags 做兜底，不暴露裸 `{tags.*}` 占位。

**根因方向**：提示词格式模板（`promptFormats`，如 Midjourney）含 `{tags.style}` 等占位，依赖 item 的 tags；复刻流程（download+analyze，跳过了 note/总结）没生成 tags → 占位裸露。需二选一：① 复刻流程补一步 tags 生成；② 提示词渲染层对缺失 tag 占位做剔除/兜底。

**证据**：截图（当前提示词文本带 `{tags.*}` + 「内容标签：暂无标签」）。
