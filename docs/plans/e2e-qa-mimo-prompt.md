## 0. 方法

- **启动**：项目根 `./dev.sh` 重启前后端（必须，别手动单起/绕 CORS）。
- **工具**：`webapp-testing` skill（启 dev server + 前端流程 + DOM + 截图 + console），`playwright` mcp（浏览器自动化）。截图存 `docs/e2e-test/screenshots/`（已有先例）。
- **每模块查 4 面**：① 流程通不通 ② 数据对不对（逻辑）③ UI（布局/交互/空态/暗色）④ 报错体验。
- **记录**：bug → `docs/e2e-test/BUGS.md`（模块 / 现象 / 复现步骤 / 截图路径 / 严重度 P0 阻断·P1 影响·P2 瑕疵）。**先全测记录，不修**（修复另轮）。
