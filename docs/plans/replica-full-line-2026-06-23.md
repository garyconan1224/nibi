## 二、用户已拍板的设计(2026-06-23 讨论结论)

1. **复刻二级 = 3 个**(用户确认):
   - **复刻提示词**:关键帧 → 画面描述提示词(给 Midjourney/可灵/Sora 等 AI 生图生视频用)
   - **拉片分析**:逐镜头拆解镜头语言(景别/构图/运镜/转场/节奏/色调),给学习模仿拍摄
   - **竞品/对标分析**:内容结构/爆点/钩子/节奏拆解(短视频运营向)
   - (不做"文案/脚本提取")
2. **处理步骤随二级动态变化**(像笔记的动态 steps,不能复用笔记的固定流程):
   - 复刻提示词:`下载 → 关键帧截图 → 画面提示词提取 → 完成`(✅ 用户确认)
   - 拉片分析:`下载 → 截帧 → 镜头语言分析 → 拉片报告 → 完成`(待细化)
   - 竞品对标:`下载 → 转写+截帧 → 结构/爆点分析 → 报告 → 完成`(待细化,注意需转写)
3. **复刻结果页**:用 `video_detail` / `image_result`(逐帧+提示词),按 3 种二级可能呈现不同(待设计)。
4. **复刻项目入口**:结果页右上放「复刻」按钮 → 打开**复刻项目列表**(按 `intent='replica'` 筛选,复用现有列表组件)。用户原话:"快速打开做过复刻的项目,看里面内容"。
