## 一、数据模型（调研结论 · 关键）
`summaries` 本就是**两层**（`services/summaries.ts:5`）：
- `template` = **总结风格**（精简/详细/教程…）
- `version` = 同一风格重新生成的**版本**（v1/v2…）
- `listSummaries` 后端就是「按 template 分组、version 排序」。

**问题根源**：现在 UI 把两层**拍平**了——顶栏「版本▾」(`index.tsx:933`) 和右侧「换总结」面板
(`SummariesTab`) 都扁平列「精简·v1、详细·v1」，同一份数据两处显示 → 10 条「两个总结/两个 V0」。

## 二、现状各处
- 顶栏「版本▾」(`:933-984`)：扁平列 summaries、点击切换。
- 右侧「换总结」面板：`SummariesTab`（537 行）——列表/应用/双击改名/删/新建(NewSummaryModal)。
- AI工具菜单(`:1082-1116`)：「原文对照」(聚焦左列字幕) +「重新生成」(新建总结)。
- 源 md：右侧操作区按钮 → `SourceMdModal` 弹窗（`:570`）。
- 左列：视频播放器 + 字幕 `LNTranscriptPanel`（`:37` import，`:1117` 左列）。

## 三、重组方案（用户已拍板）
1. **顶部两层下拉**（替换现顶栏「版本▾」）：
   - **[总结风格▾]**：列 template（精简/详细/教程），选当前风格 + **新建风格**（NewSummaryModal）。
   - **[版本▾]**：列**当前选中风格**下的 version（v1/v2），切换。两个下拉联动。
2. **删右侧「换总结」面板**（`SummariesTab` 从右侧移除）：其新建/改名/删/应用**迁入顶部两下拉**——不可丢功能。
3. **AI工具菜单**：删「原文对照」(左列已有字幕) + 删「重新生成」(并入 总结风格▾ 新建)；保留占位项。
4. **源 md 合并左列**：源 md 并入左列字幕区（`LNTranscriptPanel` 加「源 md」入口/tab，原文+字幕一处看）；右侧操作区随之撤掉（换总结已移顶部、源md已移左列）。
5. 导出菜单「原文对照(txt/srt)」**保留不动**。

  新建/改名/删/应用迁入顶部下拉 + 删右侧面板。**功能不可丢、命名理清**（风格名 + 版本号分开显示）。
