> 给 mimo 的测试执行卡。playwright 自动驱动 + 截图存证，逐用例产出结果报告。
> 状态：ready ｜ 性质：**只测不改**（发现 bug 只记录，不顺手修）｜ 预计很长（约 9 个完整 pipeline，带图单个 10+ 分钟）

## 目标

覆盖 **5 个输入源 × 带图/不带图**，从「添加素材」一路跑到「结果页所有按钮」，全程 playwright 自动操作 + 关键步骤截图，产出一份逐用例的测试报告（状态 / 耗时 / 产物 / 错误）。

## 前置准备

1. 启动：项目根 `./dev.sh`（前端 5177 / 后端 8000），确认两端都起来。
2. **带图测试的视觉模型**：用 SiliconFlow（已配 key；Anthropic 未配 key 不可用）。**带图用例的视觉模型统一选 `Qwen/Qwen3-VL-30B-A3B-Instruct`**（MoE 架构，激活仅 ~3B，速度接近 8B 但画面理解接近 30B，最适合批量截帧描述）。省钱可用 `Qwen/Qwen3-VL-8B-Instruct`；**不要选 Thinking 版**（帧分析不需要长推理，又慢又费 token）。
3. B站若要拿 AI 字幕需配 `SESSDATA` cookie（没有就只能用稿件自带字幕或走 ASR，照样记录）。
4. **新建一个专用工作空间**（名字如「E2E测试」），不要在现有工作空间里测，避免污染数据。
5. 截图存 `frontend/test-results/e2e/`，报告写 `docs/test-reports/e2e-2026-06-12.md`（目录不存在就建）。

## 执行批次（分 3 批，每批一个 mimo 会话；结果都汇总到同一份报告）

**批 1 — 先跑通主干**：B站①（带图 + 不带图）
- 这批把**结果页所有按钮 checklist 全测透**，确认主流程无误，再往下铺。

**批 2 — 铺视频平台**：B站② + 抖音 + 本地视频（各带图 + 不带图）
- 按钮抽测关键项（三视图 / 时间戳跳转 / 换总结 / 问 AI / 有无配图）。
- 本地视频需用户补路径；没补就先跳过，留到批 3。

**批 3 — 图文 + 可选 + 汇总**：小红书 + YouTube(可选) + 补测本地
- 小红书图文单跑（已知不完整，只验「不崩」）。
- YouTube 用户未给，mimo 能找到合适的就测，否则标"未测"。
- 汇总全部结果，出总览。

> **每个用例都跑到底**，单个完成**立刻写进报告**（别攒到最后，防中途崩了全丢）。

## 每个用例的标准流程（playwright）

1. 工作空间页 → 点「添加素材」打开弹窗（用 `browser_snapshot` 找按钮）。
2. 输入框填链接 / 本地路径 → 等 sniff 自动识别类型（截图：识别结果）。
3. 配置：
   - **带图用例**：打开「配图」开关，视觉模型选「系统默认」或一个具体 vision 模型；截帧模式默认 `auto`（其中**至少挑 1 个用例试 `manual` 模式**并记录差异）。
   - **不带图用例**：关闭「配图」开关。
4. 点「提交 / 开始」→ 截图（提交瞬间）。
5. **等待完成**：导航到该 item 的 `/note` 页（或看任务面板该项状态），用 playwright 轮询/`browser_wait_for` 等笔记正文出现；**带图超时给到 20 分钟**；失败（出现错误态）也算一种结果，照记。
6. 结果页加载后截图（全貌）。
7. 逐个试按钮（见下方 checklist），每个关键交互后截图。
8. 把该用例结果写进报告（模板见下）。

## 结果页按钮 checklist（视频用例逐项验证）

- [ ] 三视图 tab：**所见即所得 / md格式 / 源 md** 切换正常、内容一致
- [ ] 时间戳跳转：用 `browser_click`（真实 Playwright 操作，走浏览器完整 mousedown→click 事件链）点**非 0 秒** `[mm:ss]`，所见即所得 **和** md格式 都要验；记录 data-sec、点击前后 `video.currentTime`、`video.duration`、`video.readyState`，断言 currentTime 变到目标秒数（±1s）。**禁止在 `browser_evaluate` 里用 `el.click()` / `dispatchEvent` 当作真实点击证据**（evaluate 只读状态/辅助断言，不触发 ProseMirror 等框架依赖的 mousedown 前置状态）
- [ ] 换总结：打开 6 模板弹窗 → 选一个不同模板 → 生成成功
- [ ] 导出：md 导出 + Obsidian 导出，文件能下出来
- [ ] 截图插入：按钮能把截图插进当前光标
- [ ] 问 AI：发一个问题，有流式回答返回
- [ ] 字幕区联动：点字幕条目能跳到视频对应时间
- [ ] **带图用例**：总结正文里确实有插图；**不带图用例**：纯文字无图
- [ ] 时间戳真实性：抽查 2 个 `[mm:ss]` 是否对得上视频内容（不是全 00:00）

## 红线

- **只测不改**：发现 bug 只在报告里记录（现象+报错+截图），**不要顺手改代码**。
- 只新建测试数据，**不删**现有工作空间 / item。
- 某用例失败 → 记录完整错误后**继续下一个**，不要因为一个失败中断整轮。
- 不 commit 任何东西（这是测试，截图和报告是产物，是否留档由用户定）。
