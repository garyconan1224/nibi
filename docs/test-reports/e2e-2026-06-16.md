## ④ 遇到的问题

1. **API 字段名注意**：workspace items 使用 `item_id`（非 `id`）和 `related_task_ids`（非 `task_id`）。skill 模板中的字段名 `items[].status` 正确，但定位 item 时需用 `item_id` 字段。
2. **~~带图用例 img 渲染差异~~（已勘误 2026-06-17）**：~~API 确认 note_body 含 7 张图片 markdown，但 WYSIWYG 视图 DOM 中未见 `img` 元素~~。**复核结论**：WYSIWYG ProseMirror DOM 实测 img=7（均 514×289px，naturalWidth=1280），图片正常渲染。E2E 初测 `img=0` 是断言时机问题（可能在视图切换过程中采样），非产品 bug。B站② 带图 item 同样验证 img=4 正常。
3. **不带图用例耗时比预期长**：预估 2-5 分钟，实际 ~11 分钟。主要耗时在 LLM 生成 standard 总结（Whisper 转录约 5 分钟 + LLM 总结约 3 分钟 + 各步骤排队/开销）。可接受范围。
4. **任务中心显示「5 失败」**：底部任务栏显示历史失败任务（非本次测试产生），未报错。

1. **小红书 sniff 分类为「视频」非「图文」**：短链 `xhslink.com` 被识别为「视频」类型，走完整 video pipeline（含 transcribe + analyze）。该条内容含视频配音，分类可接受，但与预期「图文」不同。需确认：是否所有小红书链接都走 video pipeline，还是只有含视频的笔记才这样。
2. **~~时间戳 chip click seek 不生效~~（已勘误 2026-06-17）**：~~芯片元素存在且可 click，但 `video.currentTime` 始终为 0~~。**复核结论**：Playwright 原生 `page.locator('.note-ts-chip[data-sec="33"]').click()` 可正常 seek（0→33），md 格式 chip、字幕条目均正常；直接 `video.currentTime=30` 也可成功。原 E2E 失败原因是脚本使用 JS 合成 click（`el.click()` / `dispatchEvent`），绕过了 ProseMirror 的 `mousedown→click` 完整事件链，导致 `handleClick` 内部状态未初始化。非产品 bug，是测试操作方式问题。详见 2026-06-17 Playwright 复核记录。
3. **~~WYSIWYG 视图不渲染图片~~（已勘误 2026-06-17）**：~~用例3 带图笔记中，API 确认 `![...]` 存在 4 处，但 WYSIWYG DOM 中 img 标签仅 3 个~~。**复核结论**：WYSIWYG ProseMirror DOM 实测 img=4 均正常渲染，与此前批次 1 勘误一致（断言时机误报，非产品 bug）。
