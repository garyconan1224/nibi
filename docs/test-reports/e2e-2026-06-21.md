## 发现的 Bug

### Bug #1（中等）：`generate-note` 端点不支持本地文件路径

- **表现**：UI「开始生成」→ `generate-note` 端点返回 `400: "URL host cannot be empty"`
- **影响**：`generate-note` 是为网络链接设计的端点，不处理本地文件。正常"上传本地文件"的 UI 入口应走 `uploadWorkspaceItem → startItemPipeline`，但该路径受 Bug #2 制约。
- **根因**：`generate_note()` 无条件调用 `_validate_network_url()`，不区分 `source=url` vs `source=local`
- **位置**：`backend/app/routes/workspaces.py:1921`
- **建议修复**：在 `generate_note()` 开头判断，若输入为本地路径（非 http/https），返回明确错误或 redirect 到本地文件处理流程

### Bug #2（严重 → 已修复路由，下载仍有 Bug #3）：本地文件正常入口（/start）不触发 note 生成

- **表现**：通过 `/items` + `/start` 创建的本地文件 item，`analyze` 任务正常完成（截帧+VLM），但 `note` 任务不触发
- **影响**：**这是本地文件无法生成笔记的主因。** 正常 UI 上传本地文件走 `uploadWorkspaceItem → startItemPipeline`，但 `/start` 对 local source 只触发 `analyze`
- **根因**：`_bridge_to_pipeline_payload()` 对 local source 只返回 `("analyze", payload)`
- **位置**：`backend/app/routes/workspaces.py:1833-1849`
- **证据**：Case B/D workspace 均只挂了 `analyze-*` task_id，`frames=1`、`transcript=0`、`note_body=0`，无 note 任务
- **修复状态**：✅ 路由已修复（commit `1225566`），`_bridge_to_pipeline_payload` local 分支改为返回 `("note", payload)`。E2E 重测确认 task_type=note、4 步 pipeline 正确配置。**但 download 步骤暴露新 Bug #3，B/D 仍无法跑通。**

### Bug #3（严重）：本地视频下载步骤查找目录错误（Bug #2 修复暴露）

- **表现**：Bug #2 修复后，本地视频正确路由到 note pipeline，但 download 步骤立即失败：`"下载失败: 本地视频未找到（['xxx.mp4']）"`
- **影响**：**本地视频即使路由正确也无法生成笔记。** download 步骤失败 → 后续 transcribe/analyze/note 全部跳过。
- **根因**：上传端点 `upload_item()` 将文件保存到 `{workspace_root}/{filename}`（`workspaces.py:1556`），但 `_download_note_source` 本地分支调用 `find_videos(get_workspace_videos_dir(workspace_id))`（`pipeline_tasks.py:1334`），查找 `{workspace_root}/videos/` 子目录——该目录不存在。
- **位置**：
  - `backend/app/services/pipeline_tasks.py:1334` — `find_videos(project_video_dir)` 查找 `{workspace_root}/videos/`
  - `backend/app/routes/workspaces.py:1556` — 上传保存到 `{workspace_root}/`
- **证据**：
  - workspace `9f53d223` 目录下文件：`三代封神_那四代呢_大疆Pocket_4首发体验.mp4`（根目录，无 `videos/` 子目录）
  - task `note-374f1711b6e2` payload: `source_type=local`, `video_basenames=["三代封神_那四代呢_大疆Pocket_4首发体验.mp4"]`
  - task error: `"下载失败: 本地视频未找到（['三代封神_那四代呢_大疆Pocket_4首发体验.mp4']）"`
- **建议修复**：`_download_note_source` 本地分支同时检查 `project_video_dir` 和 `get_workspace_root(workspace_id)`，或在上传时将视频文件复制/移动到 `videos/` 子目录

### 附带发现

- **前端 processing 页轮询 `/pipeline/tasks/{id}` 返回 404**：后端 `GET /pipeline/tasks/{task_id}` 端点实际存在于 `pipeline.py:111`。E2E 中看到 404 是因为本次通过手动 API workaround（非正常 UI 流程）启动处理，task_id 与前端 processing 页轮询使用的 id 不匹配或不存在。不影响最终结果（自动跳转结果页），但 processing 页进度条卡在 0%。
- **`/start` 启动的 item 前端无法正确导航到 processing 页**：正常 UI 流程由 `Composer.tsx` 直接 `navigate(/processing/${task_id})`；本次 E2E 手动 API 调用绕过了 UI 提交流程，前端没有拿到 task_id，因此无法自动跳转。
