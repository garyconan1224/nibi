## 模块 1：任务系统（顶层架构）

### 1.1 核心概念

- 一个**任务** = 一个独立工作空间
  - 代码层叫 `workspace`，UI 文案统一叫"任务"
- 用户可以**同时拥有多个任务**，任务列表页平铺展示，支持搜索切换
- 一个任务里**素材类型混搭**：视频 + 音频 + 图片 + 文字 任意组合，数量不限
- **去掉历史 `project` 这一层**——`WorkspaceRecord.project_id` 字段清理掉，workspace 自治

### 1.2 任务状态

| 状态 | 含义 |
|---|---|
| `active` | 任务存在，可以添加/查看素材，**不一定全部分析完** |
| `processing` | 当前有素材正在跑分析任务 |
| `analyzed` | 任务内所有素材分析都完成（**注**：分析完成 ≠ 任务结束，后续还能做复刻、AI 对话、生成新内容） |
| `archived` | 用户主动归档 |
| `trashed` | 软删除（进入垃圾桶，主列表不显示） |

### 1.3 任务字段

- **基本信息**：任务名 / 创建时间 / 更新时间 / 状态
- **背景信息（5 字段，全部可选填）**：
  - 内容类型（课程 / 会议 / 宣传片 / Vlog / 访谈 / 纯音乐 / 其他）
  - 参与人物（自由文本，例："Hugo · 影视飓风"）
  - 主题背景（自由文本，例："Q2 数码产品开箱评测"）
  - 专有名词（用于提升 Whisper / LLM 识别准确率）
  - 分析目的（复刻参考 / 竞品分析 / 内容学习 等）
- **素材列表**（每个素材类型不同，详见模块 4-7）
- **收藏夹**（item_id 列表，用于后续复刻清单）
- **提示词版本栈**（每个素材的提示词迭代历史）
- **任务级 LLM 对话历史**（独立文件存储）

### 1.4 任务操作

| 操作 | 说明 |
|---|---|
| 新建任务 | 弹窗输入任务名，创建空任务 |
| 重命名 | 任务详情页可改名 |
| 编辑背景信息 | 模态弹窗逐字段编辑 |
| 添加素材 | 详见模块 2 |
| 软删除 | 进入垃圾桶，**不立即删素材文件** |
| 从垃圾桶恢复 | 恢复到原状态 |
| 彻底删除 | 在垃圾桶里清空或单个删——**物理删除任务记录 + 所有关联素材文件**（下载的视频、截帧、字幕、生成结果都删） |
| 导出复刻工作包 | 详见模块 8 |

**垃圾桶入口位置**：放在「设置」页里，不污染主列表。

### 1.5 任务级 LLM 对话

- **对话面板** 顶部有一排「上下文素材」多选 chip
- **默认行为**：用户必须选 ≥ 1 个素材作为上下文（避免漫无目的的提问）
- **快捷按钮**：「全任务上下文」一键勾选所有素材
- **token 兜底**：选完后内容仍超出模型上下文时，后台自动 RAG 检索相关片段，界面顶部标注「上下文已自动精简」
- **对话历史**：按任务独立存储（`data/chats/<workspace_id>.jsonl`，沿用现有 chat 路由设计）

**典型业务场景**（写进 spec 帮助后续设计师/开发理解）：

| 场景 | 选几个素材 | 典型问题 |
|---|---|---|
| 跨素材对比 | 2-3 个 | "这两个视频开头风格有什么不同？" |
| 单素材聚焦 | 1 个 + 时间范围 | "这个视频 0-10 秒的提示词怎么优化？" |
| 跨素材归纳 | 3+ 个同类 | "这 3 张海报色调共同点是什么？" |
| 全任务洞察 | 全选 | "我这个任务里所有素材最高频的标签是什么？" |

### 1.6 「AI 导演」（复刻功能）暂定策略

- **当前阶段不做**——产品需求还没想清楚，先记录方向
- **未来定位**：独立的一级模块，**不再放进 Taskboard 子标签**（避免功能重复）
- **预期包含**：收藏帧（复刻清单）/ 提示词版本 / A/B 对比 / 风格 DNA / AI 对谈
- **扩展方向**：将来可能接入可灵 / 即梦 等视频图片生成模型 API，实现一键复刻
- **过渡期做法**：当前阶段任务里"能收藏帧 + 能存提示词版本"就行，对比/风格报告/生成对接 都延后

## 模块 2：输入层

### 2.1 入口与整体交互

**「添加素材」模态——4 步合一**（采用设计稿 UI 形态，新手不用跳多个页面）：

```
① 素材类型 ──→ 视频 / 音频 / 图片 / 文字
② 输入源   ──→ URL 输入框 + 上传按钮 + 拖拽区
③ 勾选分析任务 ──→ 按所选类型显示对应任务项
④ 背景信息（折叠可选）──→ 5 字段，提交时注入到所有 AI 调用
```

**交互优化（重要）**——「混合方案」：
- 用户**粘贴 URL 后**，系统自动识别平台和内容类型，**自动回填 ① 素材类型**（用户不用先选）
- 用户**拖入本地文件后**，根据 magic bytes 识别真实格式，**自动回填 ① 素材类型**
- 用户也可以**先点 ① 选类型再粘贴**（两种入口都通）

**支持的输入**：
- **网络链接**：B 站 / 小红书 / 抖音 / 快手 / YouTube / 通用网页文章
- **本地文件**：视频 / 音频 / 图片 / 文字（pdf/docx/txt/md）
- **直接粘贴文本**：仅文字类型

### 2.2 链接处理流程

```
用户粘贴 URL
  ↓
URL 格式验证 + 平台识别（域名匹配）
  ↓
内容类型检测
  ├─ 单一类型（纯视频/纯文章）→ 自动回填素材类型，继续
  └─ 混合类型（如 B 站视频可下视频/音频/同时）→ 弹窗让用户选下载哪些
  ↓
（视频）画质：默认走「设置」页的全局默认（首次默认最高）
  ↓
yt-dlp 执行下载 + 实时进度
  ↓
完成 → 原文件可单独导出
```

**画质策略**（Q2 决议）：
- **默认**：最高画质
- **可配置**：在「设置」页有"默认下载画质"全局设置（最高 / 1080p / 720p / 仅音频）
- 添加素材模态里**不显示画质选项**，简化新手交互；高级用户去设置里调

**混合类型弹窗**（Q3 决议）：
- B 站等支持视频/音频分离下载的平台，**保留这个弹窗**
- 弹窗选项：下载完整视频 / 仅下载音频 / 同时下载两者（各生成一个素材）

**异常处理**：
- 下载失败（版权 / 地区 / 网络）→ 明确提示原因，支持重试
- 平台不支持 → 尝试通用网页抓取，仍失败则提示

### 2.3 本地文件处理流程

```
用户拖拽或选择文件
  ↓
文件大小及完整性校验（本地版无大小限制；损坏文件 → 提示重传）
  ↓
格式识别（magic bytes 判断真实格式，不信赖后缀）
  ↓
格式判断
  ├─ 已支持格式 → 直接进入下一步
  └─ 不支持格式 → 后台自动转码，用户无感知（显示进度）
  ↓
读取完成 → 原文件可单独导出
```

### 2.4 批量上传（Q5 决议）

- **URL 一次只能粘贴一个**（强制用户一个个加，避免误操作）
- **本地文件支持多选 / 拖拽多个**——一次最多 N 个（N 建议 20，可在设置调）
- 多文件上传后，每个文件**独立成为一个素材**，分别进队列
- 多个素材的任务勾选**共用同一次模态的 ③ 勾选项**（用户填一次，应用到所有）

### 2.5 文本素材的输入框（Q6 决议）

- 文本输入框**保持单行 / 小尺寸**外观（不变形为大 textarea）
- 用户粘贴大段文字时：
  - 输入框内只显示**首行可见部分**（其余字符隐藏，但内部完整保留）
  - 右侧显示「已粘贴 N 字」的提示
  - 不允许用户在框内编辑，只能"清空"
- 分析完成后，进文字详情页才看到完整文本

### 2.6 任务勾选默认（Q7 决议）

**智能默认**：根据素材类型，预先勾选最常用的分析项：

| 素材类型 | 默认勾选项 |
|---|---|
| 视频 | 画面提示词生成 ✓ / 视频文案总结 ✓ / 字幕导出 ✓ / 音乐分析 ✗ |
| 音频 | 人声转写+总结 ✓ / 字幕导出 ✓ / 音乐分析 ✗（除非 VAD 检测无人声自动切换） |
| 图片 | 内容识别 ✓ / OCR ✓ / 提示词生成 ✓ / 联想总结 ✗ |
| 文字 | 摘要+要点+金句 ✓ / 改写 ✗ / 翻译 ✗ / 多文对比 ✗ |

**用户可在「设置」页自定义"我的默认勾选偏好"**——下次添加素材时按用户设的默认走，不按系统智能默认。

### 2.7 队列管理（Q4 决议）

- **不做独立的"批量队列面板"一级模块**
- 任务队列状态进**任务中心的「队列」子标签**
- 该子标签提供：
  - 整体进度（N 个素材在跑 / M 个排队 / K 个失败）
  - 单素材进度条 + 当前步骤名
  - 单素材操作：取消 / 失败重试
  - **不做拖拽排序优先级**（对新手价值不大）

### 2.8 共用后续逻辑

```
两条路径汇合
  ↓
统一内容类型路由（视频/音频/图片/文字 → 四大分支）
  ↓
进入对应分支的处理流程（详见模块 4-7）
```

## 模块 3：前置配置

> **重要**：前置配置「散落在三处」，各司其职——**添加素材模态（粗）/ Preflight 抽屉（细）/ 设置页（默认值）**。

### 3.1 三层配置的职责划分

| 层 | 入口 | 干什么 | 何时用 |
|---|---|---|---|
| **设置页** | 侧边栏 → 设置 | 设全局默认（模型 / 画质 / 任务勾选偏好 / 子参数默认值） | 用户初次配 + 偶尔调 |
| **添加素材模态** | 任务中心 → 添加素材 | 加新素材时的粗粒度配置（类型 + 输入源 + 勾选项 + 背景） | 每次加素材 |
| **Preflight 抽屉** | 素材卡片 → ⋯ → 编辑 preflight | 单素材的精细子参数（截帧模式、Whisper 语言、提示词格式等） | 加完素材想细调 |

**三层兜底关系**：素材实际跑分析时 = **Preflight 抽屉值 ?? 添加素材模态值 ?? 设置页默认值**

### 3.2 背景信息——任务级与素材级

- **任务级背景** 存于 `WorkspaceBackground`，5 字段，建任务时填一次
- **素材级背景** 是任务级的**覆盖层**——加新素材时，模态 ④ 默认继承任务背景，用户**可改可不改**
  - 不改 → 该素材沿用任务级背景
  - 改 → 该素材保存自己的背景，AI 调用时优先使用素材级
- 素材级背景在 Preflight 抽屉里也能继续改

### 3.3 模型选择——三层兜底

- **设置页 → 模型与渠道** 里设全局默认（视觉模型 / 文本模型 / 视频模型）
- **添加素材模态**：默认按全局值走，不让用户选（简化新手）
- **Preflight 抽屉**：用户可临时改本素材的模型选择
- 任何一层未配置时往上一层兜底

### 3.4 任务勾选——粗粒度 + 细粒度分层

**添加素材模态 ③（粗粒度）**：
- 只显示一级勾选项（"画面提示词生成 ✓" / "视频文案总结 ✓" 等）
- 子参数全部使用默认值（来自设置页 / 系统智能默认）

**Preflight 抽屉（细粒度）**：
- 展开所有子参数（按素材类型不同）：

| 素材类型 | 一级勾选项 | 子参数（示例） |
|---|---|---|
| 视频 | 画面提示词 / 文案总结 / 字幕导出 / 音乐分析 | 截帧模式（按秒/AI 镜头分析）、间隔秒数、最大帧数、镜头取帧数（2/3）、总结路径（字幕/合并/视频模型）、总结深度 |
| 音频 | 转写+总结 / 说话人音色 / 字幕导出 / 音乐分析 | Whisper 语言、说话人分离开关、Suno/Udio 格式开关 |
| 图片 | 内容识别 / OCR / 提示词 / 联想 / 多图对比 | 提示词格式（MJ/SD/JSON）、联想方向（用途/设计/竞品/情绪） |
| 文字 | 摘要+要点+金句 / 联想 / 改写润色 / 翻译 / 多文对比 | 摘要长度、联想方向、改写风格、翻译目标语 |

### 3.5 设置页结构（重组后 7 页）

砍 9 → 7，去重 + 补漏：

| 页面 | 内容 |
|---|---|
| **关于 About** | 版本、开源信息、致谢 |
| **模型与渠道** | 合并原 ProvidersManagement + ModelManagement。Provider 列表（API Key + Base URL）+ 各 Provider 下的模型分类（视觉/文本/视频）+ 全局默认模型选择 |
| **下载与画质** | 下载工具（yt-dlp 等）、默认画质（最高/1080p/720p/仅音频）、混合类型默认行为 |
| **分析默认偏好** | 吸收原 ScreenshotPage + TranscriberPage + PromptFormat。包含：截帧默认参数 / Whisper 默认参数 / 提示词模板 / **任务勾选默认偏好**（用户自定义"我添加视频时默认勾哪些"） |
| **网络** | 代理 / 超时 / DNS（国内用户访问 OpenAI 等需要） |
| **部署监控** | 沿用现有 DeployMonitor |
| **任务垃圾桶** | 软删除任务列表，每条带"恢复 / 彻底删除"操作，顶部"清空垃圾桶（含所有原素材文件）" |

**移除**：
- ~~PromptFormat 独立页~~ → 并入「分析默认偏好」
- ~~ScreenshotPage 独立页~~ → 并入「分析默认偏好」
- ~~TranscriberPage 独立页~~ → 并入「分析默认偏好」
- ~~ModelManagement 独立页~~ → 并入「模型与渠道」

### 3.6 模型 Provider 配置规格

- 支持格式：**OpenAI 兼容** / **Anthropic 原生** / **本地 Ollama**
- 单个 Provider 字段：名称 / Base URL / API Key / 备注
- 一个 Provider 下可挂多个模型，每个模型标签：视觉 / 文本 / 视频（用于全局默认选择匹配）
- **API Key 加密存储**（不明文落盘，前端只显示掩码）
- 支持"快速选择"预设：OpenAI / Anthropic / Ollama / DeepSeek / Qwen / 自定义

## 模块 4：视频分支

### 4.1 可勾选的分析任务（4 项）

| 任务 | 说明 | 默认勾 |
|---|---|---|
| 画面提示词生成 | 截帧 → 视觉模型 → MJ/SD/JSON 提示词 | ✓ |
| 视频文案总结 | 三条路径选一 | ✓ |
| 字幕导出 | .srt 文件 | ✓ |
| 音乐分析 | BPM / 调性 / Suno-Udio 提示词（联动模块 5） | ✗ |

### 4.2 画面提示词生成

#### 4.2.1 截帧模式（用户在 Preflight 抽屉选）

**模式 A：按秒截帧**
- 参数：每 N 秒一帧（默认 5）/ 最大帧数（默认 100）
- 用 ffmpeg 均匀截取
- **后置自动过滤相似帧**（用户无感知），算法：相邻帧像素差异 / phash 比较

**模式 B：AI 镜头分析（默认）**
- 用 **PySceneDetect**（开源场景检测库）自动识别镜头切换点
- 弹窗展示「检测到 N 个镜头」让用户确认
- 用户选每镜头取几帧：
  - **2 帧模式**：首帧 + 尾帧（适合简单运镜）
  - **3 帧模式**（默认）：首帧 + 中间帧 + 尾帧（适合复杂镜头）
- 不再过滤相似帧（按切换点截不会重复）

#### 4.2.2 视觉模型分析

每帧送入用户配置的视觉大模型，输出 5 维度：
- 主体 / 风格 / 光线 / 色调 / 构图

**异常处理**：API 失败自动重试 3 次，超限标注"该帧分析失败"并跳过。

#### 4.2.3 提示词输出（当前阶段：3 种格式）

| 格式 | 内容 |
|---|---|
| **Midjourney（MJ）** | 含 `--ar` / `--style` / `--s` 等 MJ 专属参数 |
| **Stable Diffusion（SD）** | 适配 SD 关键词风格 |
| **JSON** | 含 timestamp / title / prompt_mj / prompt_sd / tags 字段 |

**语言**：默认英文；Preflight 抽屉可切换中英双语。

#### 4.2.4 延后事项（不要忘记）

⏳ **视频运镜提示词** — 第 4 种格式，描述运动起点和终点（配合 AI 镜头分析的 2 帧/3 帧模式），当前阶段不做，**后续 AI 导演模块开发时一并补上**。

#### 4.2.5 自动打标签入库

提示词按 7 维度自动归类（沿用 Phase 3C 已实现的标签库）：
风格 / 光线 / 构图 / 色调 / 镜头 / 人物 / 场景

### 4.3 视频文案总结（3 条路径都做）

#### 4.3.1 字幕预处理（前置步骤）

```
检测视频是否自带字幕
  ├─ 有字幕（内嵌字幕轨 / 外挂 .srt .ass）
  │     → 直接提取字幕文本，原文使用，不清洗
  │     → 字幕文件可单独导出
  └─ 无字幕
        → 提取音频轨道 → 音频文件可单独导出
        → Whisper 转写（带时间戳）
        → 字幕清洗 → 清洗后字幕可单独导出
```

**字幕清洗规则**：
- 去语气词（嗯、啊、那个）
- 合并不必要的短句
- 修正明显的识别错误
- **专有名词修正**：把任务级背景信息中填写的"专有名词"作为参数注入清洗 LLM 的 prompt，让 LLM 在清洗时优先匹配（不修改 Whisper 本身的 initial_prompt 参数，因为后处理更可控）

#### 4.3.2 三条总结路径（用户在 Preflight 抽屉选）

**路径 1：字幕直接总结**
- 流程：字幕 → 超长文本分段 → 套视频类型模板 → LLM 总结
- 输出：标题 / 摘要 / 要点列表 / 金句
- 特点：最快、最便宜
- 适合：纯语音内容

**路径 2：音视频合并总结（默认，最详细）**
- 流程：字幕 + 截帧画面描述（复用 4.2 截帧）→ 按时间戳对齐合并 → 套模板 → LLM 总结
- 输出：摘要 / 要点 / **画面引用说明（带截图）**
- **音乐分析联动**：路径 2 下如果勾了音乐分析，音乐分析必跑

**路径 3：视频模型直接分析（最轻量）**
- 用户选视频大模型：Gemini 1.5 Pro / GPT-4o / Qwen-VL-Max 等
- 整段视频送入模型（无需截帧）
- 输出：视频简单描述（质量依赖模型）
- 界面标注"所用模型名称"

#### 4.3.3 视频类型 → 总结模板

视频类型从**任务级背景信息**的「内容类型」字段读取（不重复让用户选）：

| 内容类型 | 输出侧重 |
|---|---|
| 课程 | 知识点列表 / 重点概念 / 学习建议 |
| 宣传片 | 核心卖点 / 目标受众 / 视觉亮点 |
| Vlog | 地点 + 事件亮点 / 情绪氛围 |
| 访谈 | 受访者观点 / 金句摘录 / 话题梳理 |
| 新闻报道 | 事件背景 / 关键信息 / 时间线 |
| 其他 | 通用模板（摘要+要点+金句） |

### 4.4 视频结果交互界面（沿用设计稿）

│  顶部：导航 + 标题 + .srt/.md 导出│ 右侧浮动面板  │

## 模块 5：音频分支

### 5.1 可勾选的分析任务（5 项）

| 任务 | 默认勾 | 说明 |
|---|---|---|
| 人声转写 + 内容总结 | ✓ | Whisper + LLM |
| 说话人音色区分 | ✓ | 声纹聚类（pyannote.audio） |
| 生成字幕文件 | ✓ | .srt / .txt |
| **是否进行音乐分析** | ✗ | 用户主动勾选，无论有无人声 |
| 生成音乐提示词 | ✗（依赖音乐分析勾选） | Suno/Udio 通用格式 |

### 5.2 VAD 人声检测分流

```
音频上传 → VAD 人声活动检测
  ├─ 有人声 → 人声处理路径（5.3）
  └─ 无人声 / 纯音乐
       ├─ 用户已勾音乐分析 → 直接走音乐分析路径（5.4）
       └─ 用户未勾音乐分析 → 弹窗提示"未检测到人声，是否切换为音乐分析模式？"
            └─ 用户确认 → 启用音乐分析 + 持久化到该素材 preflight
```

### 5.3 人声处理路径

**Whisper 参数**（来自 Preflight 抽屉 / 设置页默认）：
- 语言：自动检测 / 中文 / 英文 / 其他
- 说话人分离：开 / 关（会议、采访场景推荐开）

**说话人识别**（技术选型）：
- 用 **pyannote.audio**（开源主流方案）做声纹聚类
- 输出"检测到 N 个说话人"
- 结合任务背景信息中的「参与人员」字段尝试自动匹配人名
- 弹窗让用户**确认 / 修正**说话人标签（每个说话人显示一段示例音频）

**转写输出格式**：
```
[00:00:05][张总] 大家好，今天我们讨论 Q3 战略...
[00:00:18][李总] 我觉得首先要明确目标客户...
```

**文本清洗**：
- 去语气词 / 合并短句 / 修正识别错误
- 注入任务背景的「专有名词」做修正（与模块 4 字幕清洗机制一致）

**字幕导出**：
- `.srt`：带时间戳和说话人标注
- `.txt`：纯文本（可选是否带说话人）

**总结输出（按内容类型套模板）**：

从任务级背景信息「内容类型」自动选择模板，**不让用户在此重复选**：

| 内容类型 | 输出字段 |
|---|---|
| 播客 | 摘要 / 要点 / 金句 / 话题亮点 / 嘉宾观点 |
| **会议录音** | 摘要 / 要点 / 金句 + **待办事项 / 决策记录 / 发言人小结**（会议特有） |
| 课程讲解 | 摘要 / 要点 / 金句 / 知识点 / 重点概念 |
| 演讲 | 摘要 / 要点 / 金句 / 核心论点 |
| 采访 | 摘要 / 要点 / 金句 / 受访者观点 / 问答摘要 |

超长文本自动分段处理后合并。

### 5.4 音乐分析路径

**触发条件**：
- 用户主动勾选了「音乐分析」
- 或视频路径 2 + 用户勾了音乐分析（详见 5.5）
- 或 VAD 检测无人声 + 用户在弹窗确认切换

**片段切分（技术选型）**：
- 采用**音频特征自动检测**（BPM 变化、能量变化）做切分
- 切分失败 / 检测异常时回退「固定时长 90 秒一段」
- 切分粒度：每段约 1-3 分钟，避免过细

**每段音乐 6 维度分析**：
- 风格（流行/爵士/电子/古典 等）
- 情绪（欢快/悲伤/激昂/舒缓 等）
- BPM（每分钟节拍数）
- 主要乐器
- 调性（C 大调 / A 小调 等）
- 整体氛围（温暖/冷峻/神秘 等）

**LLM 生成三类输出**：
1. 相似曲风参考（推荐风格方向）
2. 适合使用场景（广告 / Vlog 配乐 / 电影配乐 等）
3. **AI 音乐生成提示词**（一份通用格式，兼容 Suno 与 Udio——两者格式差异不大，不分开输出）

**输出 JSON 示例**：
```json
{
  "segments": [
    {
      "start": "00:00:00", "end": "00:01:30",
      "style": "indie folk", "mood": "nostalgic, warm",
      "bpm": 92, "instruments": ["acoustic guitar", "soft vocals"],
      "key": "G major", "atmosphere": "intimate, contemplative",
      "music_prompt": "indie folk, acoustic guitar, soft male vocals, 92 bpm, ..."
    }
  ],
  "overall_style": "modern indie folk with cinematic touches",
  "similar_references": ["Bon Iver", "Sufjan Stevens"]
}
```

### 5.5 与视频路径 2（音视频合并）联动

**重要变更**——不强制：

- 视频路径 2 默认**只用画面描述 + 字幕合并**，不强制跑音乐分析
- 如果用户在视频素材的 preflight 中**勾选了音乐分析**，路径 2 才合并音乐数据生成"音画提示词"
- 没勾就照原始勾选走（尊重用户选择，不强加未勾任务）

### 5.6 音频结果交互界面（沿用设计稿）

布局：
```
┌────────────────────────────────────────┬──────────────┐
│ 顶部：标题 + 时长 + 导出按钮            │ 右侧总结面板  │
├────────────────────────────────────────┤              │
│ 波形图（Waveform，播放进度跟随）        │ abstract     │
├────────────────────────────────────────┤ keyPoints    │
│ 转写区（每行：时间戳 + 说话人色块 + 文本）│ quotes       │
│                                        │ todos（会议） │
├────────────────────────────────────────┤ decisions    │
│ 音乐片段卡列表（每张含 6 维度 + 复制按钮）│ speakerSummary│
└────────────────────────────────────────┴──────────────┘
```

**核心交互**：
- 点击波形某处 / 转写行 → 跳转播放位置
- 转写当前行自动高亮
- 音乐片段卡的"复制 Suno 提示词"按钮
- 说话人色块用任务背景的人名（已匹配的）

**后续可能改的方向**（先做现状，根据使用反馈再调）：
- 音乐片段卡改成第二条时间轴对齐字幕
- 增加波形上"说话人色带"可视化

### 5.7 导出节点（绿色节点）

- 原音频文件（.mp3 / .wav）
- 提取后的字幕文件（.srt / .txt，带说话人）
- 最终总结（.md / .json）
- 音乐分析结果（.json，含 music_prompt）

### 5.8 异常处理

| 异常 | 处理 |
|---|---|
| Whisper 转空（确实无人声且未勾音乐分析） | 弹窗提示切换音乐分析 |

| 音乐特征检测失败 | 回退固定 90 秒切片 |
| 文本超长 | 自动分段处理后合并 |

## 模块 6：图片分支

### 6.1 可勾选的分析任务（5 项）

| 任务 | 默认勾 | 说明 |
|---|---|---|
| 内容识别描述 | ✓ | 视觉模型分析，6 维度 |
| OCR 文字提取 | ✓ | 中英混合 / 手写 / 竖排 |
| 画面提示词生成 | ✓ | MJ / SD / JSON |
| 内容联想总结 | ✗ | 4 维度推导 |
| 多图对比 | ✗（动态出现） | 任务内图片 ≥ 2 张时才显示，需用户主动勾 |

### 6.2 基本信息读取

- 分辨率 / 格式 / 文件大小
- **EXIF**（7 字段）：拍摄设备 / 镜头 / 快门 / ISO / 光圈 / 拍摄地点 / 拍摄日期
- **EXIF 不存在时**（截图、AI 生成图、剥离过的图）：EXIF 区域显示"无 EXIF 信息"，其他分析照常进行；不强行推断

### 6.3 四个并行任务（互不依赖，可并发）

#### 任务 1：内容识别描述

视觉模型输出 **6 维度**：
- 主体（subject）
- 场景（scene）
- 色调（tones）
- 构图（composition）
- 风格（style）
- 细节（detail）

#### 任务 2：OCR 文字提取

**引擎选型**：
- **默认 PaddleOCR**（中文场景效果优，开源）
- 用户可在「设置 → 分析默认偏好」切换为"调用视觉大模型做 OCR"（更准但贵）
- 不再使用 Tesseract（中文一般）

支持：中英混合 / 手写体 / 竖排

输出：清洗排版后的文本，可导出 `.txt`

#### 任务 3：画面提示词生成

- 复用模块 4.2 的视觉模型分析逻辑
- 输出 MJ / SD / JSON 三种格式
- 自动打 7 维度标签（沿用 Phase 3C）

#### 任务 4：内容联想总结

**4 个固定联想方向**（用户多选）：
- 用途推断（usage）
- 设计风格分析（intent）
- 竞品分析（insight）
- 情绪解读

输出 4 维度：用途 / 目标受众 / 设计意图 / 竞品洞察

> ⏳ **延后事项**（不要忘）：联想方向是否支持"用户自定义补充第 5 条"或"完全自由文本输入"——用户使用一段时间后再决定是否扩展。当前阶段先做 4 个固定选项。

### 6.4 多图对比

**触发条件**：
- 任务内图片素材 ≥ 2 张时，"多图对比" 勾选项**自动出现在 preflight 中**
- 默认不勾，需用户主动开启

**对比流程**：
1. 各图先单独完成上面的 4 个并行任务
2. 横向对比 3 维度：
   - 风格一致性（是不是同一系列）

**对比入口位置**：

## 模块 7：文字分支

### 7.1 可勾选的分析任务（5 项）

| 任务 | 默认勾 | 说明 |
|---|---|---|
| 摘要 + 要点 + 金句 | ✓ | 一次 LLM 调用同时生成三类 |
| 联想归纳 | ✗ | 4 个方向 |
| 改写 / 润色 | ✗ | 4 种风格 + 可自定义 |
| 翻译 | ✗ | 常见语言下拉 + 自由填 |
| 多文对比 | ✗（动态出现） | 任务内文本 ≥ 2 篇时显示 |

### 7.2 三种输入方式

| 方式 | 处理流程 |
|---|---|
| **网络链接** | 通用 readability 算法抓正文（去导航 / 广告 / 评论）。微信公众号 / 小红书图文等**反爬严格的平台无法保证可用**——抓不到时提示用户"该页面无法解析，请直接粘贴正文" |
| **本地文件** | 解析 .pdf / .docx / .txt / .md |
| **直接粘贴** | 无需额外解析，直接读取（沿用模块 2.5 输入框规格） |

### 7.3 PDF 图文处理（重要技术点）

**目标**：PDF 不只是抽纯文字，还要保留图片信息（图文混合的脚本、报告、产品手册都需要）。

**候选开源方案**（落地阶段评估二选一）：

| 方案 | 特点 | 推荐场景 |
|---|---|---|
| **marker**（VikParuchuri/marker）| PDF → Markdown，保留图片 + 表格，2024 年广受好评 | 通用首选 |
| **docling**（IBM 开源）| 结构化文档解析，表格识别强 | 偏技术报告 |
| **unstructured** | 通用文档解析，可单独 OCR 图片 | 备选 |
| PyMuPDF (fitz) | 底层库，自行组装 | 兜底 |

**处理流程**：
1. PDF → Markdown（用 marker / docling）
2. 提取的图片**位置保留**（占位符或 base64 嵌入 Markdown）
3. 图片**默认不分析**（作为内嵌资源保留）
4. 用户可在 Preflight 抽屉中开启「PDF 内图片走图片分支分析」选项——此时图片**作为独立素材入库**到当前任务

### 7.4 超长文本检测 & 分段

```
检测文本字数
  ├─ 未超 LLM 上下文 → 直接分析
  └─ 超出 → 按段落 / 章节自动切分
            → 各段独立处理
            → 结果合并 + 二次精炼
```

### 7.5 四个并行任务

#### 任务 1：摘要 / 要点 / 金句

- 一次 LLM 调用同时生成三类
- **摘要长度**：50 字 / 100 字 / 200 字（默认 100 字）
- 要点和金句带**原文位置标注**

#### 任务 2：联想归纳

**4 个固定方向**（用户多选）：
- 深度解读
- 观点提炼
- 趋势判断
- 行动建议

输出：观点 / 趋势 / 建议（比摘要更深层）

#### 任务 3：改写 / 润色 / 翻译

**改写风格**（4 个预设 + 自定义）：
- 正式
- 口语
- 简洁
- 丰富
- **自定义**：用户输入风格指令（如"小红书种草风"）

**翻译目标语言**：
- 下拉清单：中 / 英 / 日 / 韩 / 西 / 法 / 德 / 俄 / 葡
- 「其他」选项 → 用户自由填语言名

输出：**原文与改写/翻译结果并排对照展示**（详情页左原文右结果）

#### 任务 4：多文对比

跟模块 6 多图对比逻辑一致：
- 触发：任务内文本素材 ≥ 2 篇时，preflight 自动出现"多文对比"勾选项
- 入口：「素材」子标签 → 多选文本 → 顶部"对比"按钮
- 对比维度：观点异同 / 立场倾向 / 信息完整性 / 时间线梳理
- 输出：表格 / 报告（用户选）

### 7.6 文字结果交互界面（沿用设计稿）

**布局**（grid: 1fr 1fr 或 1fr 400px）：

```
┌──────────────────────────┬──────────────────────┐
│ 顶部：标题 + 字数 + 阅读时长│ ▼ 摘要               │
├──────────────────────────┼──────────────────────┤
│                          │ ▼ 要点列表           │
│                          │   (含原文位置跳转)   │
│   原文完整展示           ├──────────────────────┤
│   （Markdown 渲染）      │ ▼ 金句               │
│   （直接粘贴的文本也展示）│                      │
│                          ├──────────────────────┤
│                          │ ▼ 联想归纳           │
│                          ├──────────────────────┤
│                          │ ▼ 改写 / 翻译        │
│                          │   (并排对照模式)     │

```

**改写/翻译模式特殊布局**：左侧原文 + 右侧改写结果**逐段对照**（不是整篇 vs 整篇），用户可逐段复制。

### 8.2 「导出工作包」功能——**代码保留 / UI 隐藏**

经过场景分析判断，导出 zip 工作包**没有强应用场景**：

**结论（修订版）**：
- **现有代码保留**（Phase 1I 已实现的前后端 zip 导出逻辑不删，作为可重启功能）
- **UI 入口隐藏**：

  - 任务中心 / 顶部都没有"导出工作包"按钮
