# 三角色协作与接力提示词

> 本文件定义当前项目的 AI 工具分工。默认工作方式是：Claude 桌面版 Code 制定计划，小米 v2.5pro 终端执行，Codex 验收审查。
> 除非用户明确要求，不要把三种角色混在同一轮里做。

## 1. 角色边界

| 工具 | 默认职责 | 默认不做 |
|---|---|---|
| Claude 桌面版 Code | 调查、制定计划、拆任务、写给小米的执行提示词 | 不直接写业务代码、不跑全项目体检、不默认开 subagent |
| Claude Code 终端 + 小米 v2.5pro | 执行明确任务、改代码、跑相关测试、commit、输出 Codex 审查提示词 | 不重新设计方向、不读全项目、不扩大任务范围 |
| Codex | 审查任务是否完成、跑必要验证、判断通过/不通过 | 默认不写新业务功能、不接手实现，除非用户明确升级 |

## 2. 默认接力链

1. Claude 桌面版 Code 做计划时，最后必须给出一段可直接复制给小米的“执行提示词”。
2. 小米完成执行时，最后必须给出一段可直接复制给 Codex 的“审查提示词”。
3. Codex 审查时，第一行必须给结论：`通过` / `不通过` / `需要补充验证`。
4. 若小米连续两轮失败、改动跨 5 个以上文件、涉及状态机/RAG/SSE/数据库/鉴权/加密/API key，再升级给 Claude 桌面版或 Codex 执行。

## 3. 禁止默认全项目模式

除非用户明确包含以下词之一，否则任何工具不得启动全项目扫描、全量通读、并行 agent 调研：

- 全项目体检
- 全量梳理
- 通读全部代码
- 并行调研
- 开多个 agent

即使命中上述词，也先说明范围和预算；默认最多 1 个 subagent，需要 2 个以上必须先问用户。

## 4. Claude 桌面版输出模板

Claude 桌面版做完计划后，结尾必须包含：

```text
给小米 v2.5pro 终端的执行提示词：

你是执行者，只按下面任务改代码。
不要重新规划，不要全项目调查，不要开 subagent。
启动只跑：
git status --short --branch
git log --oneline -5

只读和本任务直接相关的文件；超过 300 行的文件先 rg 定位再 sed 片段读取。
完成后跑相关测试，提交一个 commit。
最后回复：改了哪些文件、测试结果、commit hash，并给出可复制给 Codex 的审查提示词。

任务：
<这里写清楚目标、涉及文件/接口、验收标准、禁止事项>
```

## 5. 小米终端输出模板

小米完成任务后，结尾必须包含：

```text
给 Codex 的审查提示词：

你只做验收审查，不写业务功能。
请检查本次 commit 是否完成任务，必要时跑相关测试。
结论第一行写：通过 / 不通过 / 需要补充验证。

任务目标：
<复制任务目标>

本次 commit：
<commit hash>

重点审查：
1. <验收点 1>
2. <验收点 2>
3. <验收点 3>

已跑验证：
<测试命令和结果>
```

## 6. Codex 审查模板

Codex 审查时按这个输出：

```text
结论：通过 / 不通过 / 需要补充验证

证据：
- <代码/测试/运行证据>

问题：
- <没有问题就写“未发现阻断问题”>

建议下一步：
- <只给一个最小下一步>
```
