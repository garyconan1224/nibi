# 业务规格与运行时交互契约（Remix 版）

> 本文件由 `CLAUDE.md` §7 索引指向。**改 pipeline、前置配置、状态流转、阈值时必读**。
>
> 开发后端或前端业务流程代码时，必须严格参考并实现以下交互逻辑和默认阈值限制。
>
> ⚠️ **本文件是 SPEC.md 的"运行时镜像"**——SPEC.md 是产品需求级仲裁，本文件是代码层契约。如果两者冲突，以 SPEC.md 为准并更新本文件。

---

## 1. 全局状态机与跃迁逻辑

### 任务状态

`draft` (无素材) → `active` (至少一个素材在排队或处理中) → `done` (全部素材 done 或 failed 且用户已知会) → `archived` (归档)。

### 素材状态

`pending` → `downloading` (仅链接素材) → `probing` (探测格式/时长/字幕轨) → `processing` (跑 pipeline 阶段) → `done` 或 `failed`。

### 关键约束

素材一旦到 `done` 状态，结果直接写入数据库，**不再因前置配置变更而自动重跑**。重新运行必须由用户**显式触发新建作业**。

---

## 2. 前置配置依赖级联（重要，前端必须联动限制，后端必须校验）

### 2.1 字段上限

- 说话人最多 **12 人**，且名字限制 **16 字**
- 主题背景限制 **200 字**
- 专有名词最多 **30 个**，且每个限制 **24 字**

### 2.2 配置联动级联

- 若勾选「视频文案总结 - 路径 2 (音视频合并)」，**必须且自动勾选「画面提示词生成」**（需截帧画面），且在前端禁用取消勾选。
- 若勾选「说话人音色区分」，**必须且自动勾选「人声转写」**。
- 「生成字幕文件」**强制依赖**「人声转写」。
- 「多图 / 多文对比」仅在一次性上传素材数 ≥ 2 时可选，否则置灰禁用。

---

## 3. 视频处理分支参数

### 3.1 模式 A — 按秒截帧

- 默认间隔 **2 秒**
- 最大上限 **60 帧**
- 使用 **pHash 汉明距离 ≤ 6** 过滤相似帧

### 3.2 模式 B — AI 镜头

- 镜头切换判定 **SSIM < 0.55**
- **2 帧模式**取首尾，**3 帧模式**取首中尾

### 3.3 总结三路径

- **路径 1（字幕直接总结）**：仅通过字幕文本 LLM 生成
- **路径 2（音视频合并）**：字幕 ↔ 帧描述时间戳对齐，送大模型
- **路径 3（视频模型直跑）**：直接送视觉/视频大模型（如 Gemini 1.5 Pro / GPT-4o），不进行额外抽帧

---

## 4. 音频处理分支参数（VAD 人声检测分叉）

| 人声占比 | 处理路径 |
|---|---|
| ≥ 15% | 执行完整人声处理（Whisper ASR + 说话人声纹聚类）|
| 5% ≤ X < 15% | 弹窗/气泡提示「人声较少，是否仅分析音乐？」|
| < 5% | 默认按纯音乐路径处理 |

---

## 5. 处理阶段可跳过策略（Resiliency）

### 5.1 处理顺序

```
download → probe → frames → asr → vlm → sum → store
```

### 5.2 各阶段可跳过性

| 阶段 | 失败行为 |
|---|---|
| `download` | **失败直接终止作业** |
| `probe` | **失败直接终止作业** |
| `frames`（截帧）| **可跳过**，跳过后总结退化为路径 1 |
| `asr`（转写）| **可跳过**，跳过后总结跳过文本分析 |
| `vlm`（视觉分析）| 单帧失败不终止任务，对应帧标记「分析失败」 |
| `sum`（总结）| **不可跳过** |
| `store`（存储）| **不可跳过** |

---

## 6. 存储与自动清理策略

| 类型 | 保留期 |
|---|---|
| 截帧、临时转码文件、临时提取音频等中间产物（默认） | **14 天** |
| 中间产物（归档任务） | **3 天** |
| 原始文件 + AI 文本分析结果 | **永久保留**，直至用户手动删除任务 |
