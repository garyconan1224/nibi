# 多媒体内容分析系统 — 完整方案文档 v2.1

> 版本：v2.1｜用途：本地工具（考虑开源/云端收费两条路）｜核心场景：AI视频/图片复刻
> 供AI开发参考

---

## 一、系统概述

用户新建任务，在任务中持续添加网络链接或本地文件，系统自动识别内容类型，调用对应AI模型分析，所有结果归入任务数据库，支持跨素材提问、复刻参数导出和结果回放。

**核心定位：帮助AI视频/图片创作者快速复刻别人的成功作品。**

**部署方式：**
- 本地版：优先开发，考虑开源
- 云端版：未来方向，需要收费商业化

---

## 二、任务系统

### 2.1 任务结构

```
任务（Task）
├─ 基本信息：任务名称 / 创建时间 / 任务状态
├─ 素材列表：可持续添加，每次添加均归入同一任务
├─ 任务数据库：所有素材的分析结果统一存储
│   ├─ 提示词（按维度自动分类 + 支持手动打标）
│   ├─ 字幕文本（带时间戳）
│   ├─ 截帧图片
│   ├─ 音乐分析结果
│   ├─ 文案总结
│   └─ 参考帧收藏夹
└─ 任务级LLM对话（跨素材提问）
```

### 2.2 任务操作

| 操作 | 说明 |
|------|------|
| 新建任务 | 填写任务名称，创建空任务 |
| 添加素材 | 粘贴链接或上传文件，可多次添加 |
| 查看任务 | 素材列表 + 分析结果 + LLM对话入口 |
| 任务LLM对话 | 基于任务内所有数据提问 |
| 导出工作包 | 一键打包任务内所有复刻素材 |

### 2.3 任务级LLM对话示例

```
"这10个视频的画风提示词有哪些共同词？"
"帮我整合所有音乐分析，找出共同风格"
"哪个视频的镜头语言最值得复刻？"
"把所有金句整理成一份表格"
```

## 三、输入层

### 3.1 两种入口

| 入口 | 说明 |
|------|------|
| 网络链接 | B站、小红书、抖音、快手、YouTube、网页文章 |
| 本地文件 | 视频、音频、图片、文字文件（拖拽或点击上传） |

### 3.2 链接处理流程

```
URL输入
→ URL格式验证
→ 平台识别（域名匹配）
→ 内容类型检测（视频/音频/图文混合/纯文章）
→ 用户选择下载内容（混合内容时弹出，单一类型自动跳过）
→ yt-dlp 执行下载（画质选择：最高/1080p/720p/仅音频）
→ 下载完成 → 可直接导出原文件
```

**异常处理：**
- 平台不支持：提示用户，尝试通用网页抓取
- 下载失败（版权/地区/网络）：明确提示原因，支持重试

### 3.3 本地文件处理流程

```
文件上传（拖拽/点击/批量）
→ 文件大小及完整性校验（本地版无大小限制）
→ 格式识别（文件头magic bytes判断）
→ 不支持格式 → 后台自动转码（显示进度，用户无需操作）
→ 读取完成 → 进入统一处理流程
```

### 3.4 共用逻辑

```
两路汇合
→ 统一内容类型路由（视频/音频/图片/文字）
→ 批量任务判断
  → 单个：直接进入配置
  → 批量：进入队列管理
→ 前置配置（背景信息 + 任务勾选）
→ 路由到对应功能模块
→ 结果存入任务数据库
```

## 四、前置配置（所有模块共用）

### 4.1 背景信息填写（可选）

| 字段 | 示例 |
|------|------|
| 内容类型 | 课程、会议、宣传片、Vlog、访谈、纯音乐 |
| 参与人员 | 张总、李总 |
| 主题背景 | Q3战略会议 |
| 专有名词 | 产品名、技术术语 |
| 分析目的 | 复刻参考、竞品分析 |

### 4.2 模型配置

> 模型由用户自行添加和选择，系统不内置固定模型。

```
模型管理入口（系统设置）
├─ 添加模型：填写 API Key + 模型名称 + Base URL
├─ 支持类型：
│   ├─ 文本大模型（用于总结/归纳/对话）
│   ├─ 视觉大模型（用于图片/截帧分析）
│   └─ 视频大模型（用于视频直接分析路径3）
├─ 可添加多个模型，每次任务时选择使用哪个
└─ 支持主流格式：OpenAI API兼容格式 / Anthropic / 本地Ollama
```

**任务执行时选择模型：**
- 视觉分析用哪个模型（如 GPT-4V / Claude Vision / Gemini）
- 文本总结用哪个模型（如 GPT-4o / Claude / Qwen）
- 视频直接分析用哪个模型（如 Gemini 1.5 Pro / GPT-4o）

### 4.3 执行前任务勾选

| 模块 | 可勾选任务 |
|------|----------|
| 视频 | 画面提示词、视频文案总结（路径1/2/3）、字幕导出 |
| 音频 | 人声转写总结、说话人音色区分、生成字幕、音乐分析、音乐提示词 |
| 图片 | 内容识别描述、OCR文字提取、画面提示词、联想总结、多图对比 |
| 文字 | 摘要/要点/金句、联想归纳、改写润色、翻译、多文对比 |

## 六、视频模块

### 6.1 功能A：画面提示词生成

```
视频输入
→ 读取视频基本信息（时长/分辨率/帧率）
→ 选择截帧模式
  ├─ 模式A：按秒截帧
  │   → 用户设置：每N秒1张 + 最大帧数上限
  │   → ffmpeg执行截帧
  │   → 过滤相似帧（※仅此模式需要）
  └─ 模式B：AI镜头分析
      → 自动识别镜头切换点，展示镜头数
      → 每镜头取2帧（首+尾）或3帧（首+中+尾）
      → 直接得到帧列表（无需过滤）
→ 逐帧送入视觉大模型（用户已选模型）
  → 输出：主体/风格/光线/色调/构图（英文）
  → 失败自动重试×3
→ 汇总帧描述（按时间序）
→ 用户选择提示词风格
  → 风格：Midjourney / Stable Diffusion / 视频运镜
  → 格式：纯文本 / JSON
  → 语言：英文 / 中英双语
→ LLM精炼为提示词（用户已选模型）
→ 提示词自动打标签入库
→ 结果存入任务数据库
```

**JSON输出格式：**
```json
{
  "frames": [
    {
      "timestamp": "00:00:05",
      "thumbnail": "frame_001.jpg",
      "description": "A woman in a red dress standing near a window...",
      "prompt_mj": "woman in red dress, window light, cinematic, --ar 16:9",
      "prompt_sd": "woman in red dress, window light, cinematic lighting",
      "tags": ["portrait", "warm tones", "window light", "cinematic"]
    }
  ],
  "overall_style": "cinematic, warm tones, portrait photography"
}
```

### 6.2 功能B：视频文案总结

**字幕检测：**
```
有字幕 → 直接提取，无需清洗 → 可导出字幕文件
无字幕 → 提取音频 → Whisper转写 → 字幕清洗 → 可导出字幕
```

**三条总结路径：**

| 路径 | 方式 | 适合场景 |
|------|------|---------|
| 路径1：字幕总结 | 字幕文本→LLM | 纯语音内容，速度最快 |
| 路径2：音视频合并 | 字幕+截帧→时间戳对齐→LLM | 需画面引用，最详细 |
| 路径3：视频直接分析 | 整段视频→视频大模型 | 快速预览，最轻量 |

**视频类型与总结结构：**

| 类型 | 总结侧重 |
|------|---------|
| 课程 | 知识点、重点概念、学习建议 |
| 宣传片 | 核心卖点、目标受众、视觉亮点 |
| Vlog | 地点/事件亮点、情绪氛围 |
| 访谈 | 受访者观点、金句、话题梳理 |
| 新闻 | 事件背景、关键信息、时间线 |

### 6.3 视频结果交互界面

```
视频详情页布局：
┌─────────────────────────────────────────┐
│  内嵌视频播放器（上方）                    │
├─────────────────────────────────────────┤
│  三轨时间轴面板：                          │
│  轨道1：镜头缩略图（点击跳转）              │
│  轨道2：字幕文本（随播放高亮滚动）           │
│  轨道3：提示词高亮区间（对应镜头时段）       │
├─────────────────────────────────────────┤
│  右侧面板：                               │
│  当前帧提示词（随时间轴更新）               │
│  一键复制 / 收藏此帧                       │
└─────────────────────────────────────────┘
```

## 七、音频模块

### 7.1 人声处理路径

```
VAD人声检测（有声/无声分叉）
→ 有人声：
  → Whisper参数设置（语言/说话人分离开关）
  → 音色识别/说话人区分（声纹聚类→匹配人名→用户确认标签）
  → Whisper转写 → [时间戳][说话人] 逐句文本
  → 文本清洗 → 可导出 .srt / .txt
  → 选择输出内容 → LLM按内容类型总结
→ 无人声：自动切换音乐分析模式（若勾选）
```

### 7.2 音乐分析路径

> 触发条件：用户勾选"音乐分析"，无论有无人声都执行。

```
音乐片段切分（按时间段，标注时间戳区间）
→ 每段特征分析：风格/情绪/BPM/主要乐器/调性/氛围
→ LLM生成音乐提示词（相似推荐 + Suno/Udio格式）
→ 结果存入任务数据库
```

### 7.3 合并模式（与视频联动）

- 视频文案总结路径2：音乐分析必跑
- 有人声：转写总结 + 音乐分析同时输出
- 无人声：音乐分析 + 画面描述合并生成音画提示词

## 九、复刻专项功能

### 9.1 提示词标签库

**标签来源：自动 + 手动双模式**
- 自动：分析完成后LLM按维度自动归类
- 手动：用户可自行添加、修改、删除标签

| 维度 | 示例标签 |
|------|---------|
| 风格 | cinematic, anime, realistic, watercolor |
| 光线 | golden hour, studio light, backlight |
| 构图 | close-up, wide shot, bird's eye view |
| 色调 | warm tones, desaturated, high contrast |
| 镜头 | bokeh, depth of field, motion blur |
| 人物 | portrait, full body, facial expression |
| 场景 | urban, nature, indoor, fantasy |

**标签库用途：**
- 任务内按标签筛选素材
- 跨任务检索某个标签的所有素材
- 统计某创作者最常用哪些标签

### 9.2 创作者风格提取报告

**触发方式：手动触发 + 自动触发双模式**
- 手动：用户随时点击"生成风格报告"按钮
- 自动：任务内素材数量达到用户设定的阈值时自动生成（默认5个，可调整）

```
风格报告内容：
├─ 惯用提示词词云（高频词排名）
├─ 色彩偏好（主色调分布）
├─ 镜头习惯（常用景别/角度/运镜）
├─ 叙事结构（开场/中段/结尾节奏）
├─ 音乐偏好（常用BPM/风格/情绪）
└─ 复刻建议（AI生成时关键参数推荐）
```

### 9.3 参考帧收藏夹

- 视频播放时点击"收藏此帧"
- 图片详情页点击"加入收藏"
- 收藏帧聚合到任务"复刻清单"
- 每帧显示：缩略图 + 时间戳 + 提示词 + 备注

### 9.4 提示词迭代版本记录

```
v1（初始）→ v2（修改）→ v3（修改）→ 最终版
每个版本保存：提示词内容 / 修改时间 / 备注说明 / 对应生成结果（若上传）
支持：回退任意版本 / 版本对比 / 复用历史版本
```

### 9.5 生成结果对比原作

**对比精度前置选择（执行前用户选择）：**

| 精度模式 | 说明 | 耗时 |
|---------|------|------|
| 整体风格对比 | 色调/氛围/风格大方向 | 快（约10秒） |
| 构图细节对比 | 构图/主体/景别/空间关系 | 中（约30秒） |
| 精细像素级对比 | 细节/纹理/色彩精确度 | 慢（约60秒） |

```
用户上传生成图/视频
→ 选择对比精度
→ 系统自动与任务内参考帧配对
→ 视觉模型分析差异
→ 输出对比报告：
  ├─ 相似度评分（0-100）
  ├─ 差距分析（按所选精度维度）
  └─ 提示词优化建议
```

### 9.6 一键导出复刻工作包

> 暂不支持分享，仅本地导出。

```
复刻工作包（.zip）
├─ reference_frames/        参考帧图片（时间戳命名）
├─ prompts.json             所有提示词（MJ/SD双格式）
├─ prompts_by_tag.json      按标签分类的提示词
├─ subtitles.srt            字幕文件
├─ music_analysis.json      音乐分析（Suno/Udio提示词）
├─ style_report.md          风格报告
├─ favorites/               收藏帧及提示词
└─ README.md                使用说明
```
