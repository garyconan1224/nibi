# 多媒体内容分析系统 — 完整设计文档 v3.0

> **目的**：本文档供 Claude 设计师/开发参考，快速理解系统结构和逻辑
> **核心场景**：帮助 AI 视频/图片创作者快速复刻别人的成功作品
> **部署方式**：本地优先（考虑开源），云端版未来收费
> **使用对象**：个人创作者（暂不做团队权限）

---

## 系统总览

整个系统按三个层级组织：

1. **顶层：任务系统** — 所有内容都归属于一个"大任务"，任务内素材共享数据库
2. **中层：六大功能模块** — 输入层、前置配置、四大分析分支
3. **底层：交互与进度** — 结果交互界面、全流程进度（贯穿全系统）

---

## 第一部分：任务系统（顶层架构）

### 1.1 任务的概念

**一个任务 = 一个独立的工作空间**，包含：
- 任务基本信息（任务名称、创建时间、任务状态）
- 多个素材（视频、图片、音频、文案，可持续添加，不限数量、不限类型）
- 任务级数据库（所有素材的分析结果统一存储）
- 任务级 LLM 对话入口

### 1.2 任务的操作

| 操作 | 说明 |
|------|------|
| 新建任务 | 输入任务名称，创建空任务 |
| 添加内容 | 在任务内粘贴链接或上传文件，可分多次添加，每次添加都归入同一任务 |
| 查看任务 | 进入任务页，看到素材列表 + 各素材的分析结果 + LLM 对话入口 |
| 任务级 LLM 对话 | 基于任务内所有数据提问，跨素材分析 |
| 导出复刻工作包 | 一键打包任务内所有素材的复刻参数 |

### 1.3 任务级 LLM 对话示例

LLM 调用时会把任务内所有分析结果作为上下文注入，支持以下提问：

- 「这 10 个视频的画风提示词有哪些共同词？」
- 「帮我整合所有音乐分析，找出共同的风格特征」
- 「哪个视频的镜头语言最值得复刻？」
- 「把所有金句整理成一份表格」
- 「这批参考素材的色调有什么规律？」

## 第二部分：输入层

### 2.1 两种输入入口

**入口 A：网络链接**
- 支持平台：B 站、小红书、抖音、快手、YouTube、网页文章
- 用户操作：粘贴链接 → 系统自动识别平台和内容类型

**入口 B：本地文件上传**
- 支持类型：视频、音频、图片、文字文件
- 用户操作：拖拽上传 或 点击选择文件
- 支持批量上传

### 2.2 链接处理流程（详细）

```
用户粘贴链接
  ↓
URL 格式验证
  ↓
平台识别（通过域名匹配）
  ├─ 已知平台 → 使用对应下载策略
  └─ 未知平台 → 尝试通用网页抓取
  ↓
内容类型检测
  ├─ 单一类型（纯视频/纯音频/纯文章）→ 自动跳过下一步
  └─ 混合类型（视频+音频/图文混合）→ 弹窗让用户选择下载哪些
  ↓
画质选择（仅视频）
  - 最高画质 / 1080p / 720p / 仅音频
  ↓
yt-dlp 执行下载（实时进度条）
  ↓
下载完成 → 原文件可直接导出（绿色节点）
```

**异常处理：**
- 下载失败（版权/地区限制/网络问题）：明确提示原因，支持重试

### 2.3 本地文件处理流程（详细）

```
用户拖拽或选择文件
  ↓
文件大小及完整性校验
  - 本地版无大小限制
  - 损坏文件 → 提示重新上传
  ↓
格式识别（通过文件头 magic bytes 判断真实格式）
  ↓
格式判断
  ├─ 已支持格式 → 直接进入下一步
  └─ 不支持格式 → 后台自动转码（用户无需操作，显示进度）
  ↓
读取完成 → 原文件可直接导出（绿色节点）
```

### 2.4 共用后续逻辑

```
两条路径汇合
  ↓
统一内容类型路由（视频/音频/图片/文字）
  ↓
批量任务判断
  ├─ 单个任务 → 直接进入前置配置
  └─ 多个任务 → 进入队列管理面板
                - 显示整体进度
                - 单任务可取消、重试
                - 支持拖拽排序优先级
```

## 第三部分：前置配置（所有模块共用）

### 3.1 填写背景信息（可选，强烈推荐）

填写后的信息会注入到所有后续 AI 调用中，提升分析准确率。

**字段说明：**

| 字段 | 示例 | 影响范围 |
|------|------|---------|
| 内容类型 | 课程、会议、宣传片、Vlog、访谈、纯音乐 | 总结结构选择 |
| 参与人员 | 张总、李总、产品负责人 | 说话人识别匹配人名 |
| 主题背景 | Q3 战略会议、AI 工具评测 | LLM 上下文 |
| 专有名词 | 产品名、技术术语、人名 | Whisper 识别准确率 |
| 分析目的 | 复刻参考、竞品分析 | 总结侧重点 |

### 3.2 选择模型（用户自行添加）

> **重要**：系统不内置任何模型，全部由用户自行配置

**模型管理：**
```
系统设置 → 模型管理
  ↓
添加模型
  - API Key
  - 模型名称
  - Base URL
  ↓
分类管理
  - 视觉大模型（用于图片/截帧分析）
  - 文本大模型（用于总结/归纳/对话）
  - 视频大模型（用于视频直接分析路径3）
  ↓
任务执行时选择使用哪个模型
```

**支持格式：**
- OpenAI API 兼容格式
- Anthropic 原生格式
- 本地 Ollama

### 3.3 执行前任务勾选（核心交互）

**用户在这里决定本次要跑哪些分析项**，未勾选的完全跳过执行，节省时间和成本。

每个模块的可勾选项详见各分支说明。

## 第四部分：视频分支（完整流程）

### 4.1 进入视频分支前的勾选项

用户在前置配置时，针对视频可勾选：

| 任务项 | 说明 |
|--------|------|
| 画面提示词生成 | 截帧 → 视觉分析 → 输出提示词 |
| 视频文案总结 | 字幕 → LLM 总结（三条路径选一） |
| 字幕导出 | 转写后导出 .srt / .txt |
| 音乐分析 | 是否分析视频中的背景音乐（同纯音频的勾选项，详见 5.1） |

### 4.2 功能 A：画面提示词生成

#### 步骤 1：读取视频基本信息
- 时长、分辨率、原始帧率、文件大小

#### 步骤 2：选择截帧模式（用户决策）

**模式 A：按秒截帧**
- 用户设置参数：
  - 每隔 N 秒截 1 张图（例如每 5 秒截 1 张）
  - 最大帧数上限（例如最多 100 张）
- 系统执行：
  - ffmpeg 按设定间隔均匀截帧
  - 输出文件：frame_001.jpg、frame_002.jpg...
  - **必须执行：过滤相似帧**（比较相邻帧差异，丢弃重复画面）

**模式 B：AI 镜头分析**
- 系统执行：
  - 自动识别镜头切换点
  - 弹窗展示「检测到 N 个镜头」让用户确认
- 用户选择：每个镜头取几帧
  - **2 帧模式：取每个镜头的「首帧 + 尾帧」**
    - 适合简单运镜
    - 用于生成视频运镜提示词时，描述运动起点和终点
  - **3 帧模式：取每个镜头的「首帧 + 中间帧 + 尾帧」**
    - 适合复杂镜头
    - 中间帧描述过渡状态
    - 用于生成视频运镜提示词时，描述完整的运动轨迹
- 系统直接得到帧列表（无需过滤相似帧，因为按切换点截帧本身就不会重复）

#### 步骤 3：逐帧送入视觉大模型
- 输入：帧图片 + 系统 Prompt
- 输出维度（英文）：主体 / 风格 / 光线 / 色调 / 构图
- 异常处理：API 失败自动重试 3 次，超过则跳过该帧并标注

#### 步骤 4：汇总所有帧描述
- 按时间顺序排列
- 去除重复描述

#### 步骤 5：用户选择提示词风格（用户决策）

| 维度 | 选项 |
|------|------|
| 风格 | Midjourney / Stable Diffusion / 视频运镜 |
| 格式 | 纯文本 / JSON（含时间戳+描述+提示词字段） |
| 语言 | 英文 / 中英双语 |

#### 步骤 6：LLM 精炼为提示词
- 注入背景信息 + 用户选择的风格指令
- 生成结构化提示词

#### 步骤 7：自动打标签入库
提示词按以下维度自动归类（详见复刻专项）：
- 风格、光线、构图、色调、镜头、人物、场景

#### 步骤 8：结果存入任务数据库

**JSON 输出格式示例：**
```json
{
  "frames": [
    {
      "timestamp": "00:00:05",
      "thumbnail": "frame_001.jpg",
      "description": "A woman in a red dress standing near a window...",
      "prompt_mj": "woman in red dress, window light, cinematic, --ar 16:9",
      "prompt_sd": "woman in red dress, window light, cinematic lighting",
      "tags": ["portrait", "warm tones", "window light", "cinematic"]
    }
  ],
  "overall_style": "cinematic, warm tones, portrait photography"
}
```

### 4.3 功能 B：视频文案总结

#### 前置步骤：字幕检测

```
检测视频是否自带字幕
  ├─ 有字幕（内嵌字幕轨 / 外挂 .srt .ass）
  │     → 直接提取字幕文本（无需清洗，原文使用）
  │     → 字幕文件可单独导出
  └─ 无字幕
        → 提取音频轨道（音频文件可单独导出）
        → Whisper 转写（输出带时间戳的字幕文本）
        → 字幕清洗（去语气词 / 重复句 / 识别错误）
        → 清洗后字幕可单独导出
```

#### 三条总结路径（用户选一）

**路径 1：字幕直接总结**
- 流程：字幕文本 → 超长文本按时间段分段 → 按视频类型套模板 → LLM 生成总结
- 输出：标题 / 摘要 / 要点列表 / 金句
- 适合：纯语音内容，速度最快

**路径 2：音视频合并总结（最详细）**
- 流程：
  - 字幕文本 + 截帧画面描述（复用截帧模块）
  - 按时间戳对齐合并
  - 按视频类型套模板 + 用户设置总结深度
  - LLM 生成总结
- 输出：摘要 / 要点 / 画面引用说明（带截图）
- 注意：此路径会额外消耗截帧 + 视觉模型的时间和费用
- **音乐分析联动**：此路径下，如果用户在前置勾选了音乐分析，音乐分析必跑（详见第五部分）

**路径 3：视频直接分析（最轻量）**
- 用户选择视频大模型：
  - Gemini 1.5 Pro（长视频支持好）
  - GPT-4o（理解准确）
  - Qwen-VL-Max（中文场景强）
  - 或用户自行添加的其他视频模型
- 流程：整段视频送入模型（无需截帧）
- 输出：视频简单描述（质量依赖所选模型）
- 界面标注：所用模型名称
- 适合：快速预览

#### 视频类型与总结结构对应

| 视频类型 | 总结侧重 |
|---------|---------|
| 课程 | 知识点列表、重点概念、学习建议 |
| 宣传片 | 核心卖点、目标受众、视觉亮点 |
| Vlog | 地点/事件亮点、情绪氛围 |
| 访谈 | 受访者观点、金句摘录、话题梳理 |
| 新闻报道 | 事件背景、关键信息、时间线 |

### 4.4 视频结果交互界面（核心）

视频分析完成后，用户进入视频详情页。

**界面布局：**
```
┌─────────────────────────────────────────────┐
│                                             │
│       内嵌视频播放器（上方区域）              │
│                                             │
├─────────────────────────────────────────────┤
│  三轨时间轴面板：                            │
│                                             │
│  轨道 1：镜头缩略图（点击任意帧跳转到对应时间）│
│  轨道 2：字幕文本（随播放进度高亮滚动）       │
│  轨道 3：提示词高亮区间（对应镜头时段显示）   │
│                                             │
├─────────────────────────────────────────────┤
│  右侧浮动面板：                              │
│  - 当前帧的提示词（随时间轴自动更新）        │
│  - 一键复制提示词                            │
│  - 收藏此帧（加入参考帧收藏夹）              │
└─────────────────────────────────────────────┘
```

**交互行为：**
- 视频播放时，字幕和提示词跟着时间轴一起滚动
- 点击时间轴上任意位置（缩略图/字幕/提示词），视频跳转到对应时间
- 收藏的帧自动聚合到任务的「复刻清单」

## 第五部分：音频分支（完整流程）

### 5.1 进入音频分支前的勾选项（核心改动）

> **关键说明**：纯音频上传时和视频处理时，都在前置勾选阶段决定是否分析音乐

用户可勾选的任务项：

| 任务项 | 说明 |
|--------|------|
| 人声转写 + 内容总结 | Whisper 转写 + LLM 按内容类型生成总结 |
| 说话人音色区分 | 声纹聚类识别多个说话人，匹配人名 |
| 生成字幕文件 | 转写完成后导出 .srt / .txt |
| **是否进行音乐分析** | 用户主动勾选，无论有无人声都执行 |
| 生成音乐提示词 | 在音乐分析基础上，生成 Suno/Udio 格式提示词 |

### 5.2 VAD 人声检测（决定主流程走向）

```
VAD（Voice Activity Detection）人声活动检测
  ↓
判断人声占比
  ├─ 有人声 → 进入「人声处理路径」
  └─ 无人声 / 纯音乐 → 进入「音乐分析路径」（如果用户勾选）
                     → 如未勾选音乐分析，提示用户「未检测到人声且未勾选音乐分析」
```

### 5.3 人声处理路径（详细）

#### 步骤 1：Whisper 参数设置（用户决策）

| 参数 | 选项 |
|------|------|
| 语言 | 自动检测 / 手动指定（中文、英文、其他） |
| 说话人分离 | 开启 / 关闭（会议/采访场景建议开启） |

#### 步骤 2：音色识别 / 说话人区分（如果勾选）

```
声纹聚类
  - 自动识别音频中有几个不同说话人
  - 系统输出：检测到 N 个说话人
  ↓
结合背景信息匹配人名
  - 例如背景信息填了「张总、李总、产品负责人」
  - 系统尝试自动匹配 Speaker A→张总、Speaker B→李总
  ↓
用户确认 / 修正说话人标签
  - 界面：每个说话人显示一段示例音频和初始标签
  - 用户可修改、可跳过
```

#### 步骤 3：Whisper 执行转写

输出格式：
```
[00:00:05][张总] 大家好，今天我们讨论 Q3 战略...
[00:00:18][李总] 我觉得首先要明确目标客户...
[00:00:32][产品负责人] 从产品角度看...
```

#### 步骤 4：文本清洗

清洗规则：
- 去除语气词（嗯、啊、那个）
- 合并不必要的短句
- 修正明显的识别错误
- 注入背景信息中的专有名词进行修正

#### 步骤 5：字幕文件可单独导出（绿色节点）

| 格式 | 说明 |
|------|------|
| .srt | 带时间戳和说话人标注 |
| .txt | 纯文本（可选是否带说话人） |

#### 步骤 6：用户选择输出内容（可多选）

通用输出：
- 摘要 / 要点列表 / 金句摘录

会议场景额外输出：
- 待办事项 / 决策记录 / 发言人小结

#### 步骤 7：LLM 生成总结

- 注入：清洗后文本 + 内容类型 + 输出要求 + 背景信息
- 超长文本：自动分段处理后合并

#### 内容类型与输出对应

| 内容类型 | 特有输出字段 |
|---------|------------|
| 播客 | 话题亮点、嘉宾观点 |
| 会议录音 | 待办事项、决策记录、发言人小结 |
| 课程讲解 | 知识点、重点概念 |
| 演讲 | 核心论点、金句 |
| 采访 | 受访者观点、问答摘要 |

### 5.4 音乐分析路径（详细）

> **触发条件**：用户在前置勾选了「音乐分析」，无论有无人声都执行
> **特殊场景**：视频文案总结路径 2（音视频合并）下，如果勾选了音乐分析，音乐分析必跑

#### 步骤 1：音乐片段切分

```
按时间段切分音频
  ↓
标注每个片段的时间戳区间
  - 例：00:00:00 - 00:01:30 第一段
  - 例：00:01:30 - 00:03:15 第二段
```

#### 步骤 2：每段音乐特征分析

分析维度：
- **风格**：流行、爵士、电子、古典等
- **情绪**：欢快、悲伤、激昂、舒缓等
- **BPM**：每分钟节拍数
- **主要乐器**：钢琴、吉他、合成器等
- **调性**：C 大调、A 小调等
- **整体氛围**：温暖、冷峻、神秘等

#### 步骤 3：LLM 生成音乐提示词

输出三类内容：
1. **相似曲风参考**：推荐风格类似的方向
2. **适合使用场景**：广告、Vlog 配乐、电影配乐等
3. **AI 音乐生成提示词**：对齐 Suno / Udio 格式

#### 步骤 4：输出结构

```json
{
  "segments": [
    {
      "timestamp_start": "00:00:00",
      "timestamp_end": "00:01:30",
      "style": "indie folk",
      "mood": "nostalgic, warm",
      "bpm": 92,
      "instruments": ["acoustic guitar", "soft vocals", "light percussion"],
      "key": "G major",
      "atmosphere": "intimate, contemplative"
    }
  ],
  "overall_style": "modern indie folk with cinematic touches",
  "similar_references": ["Bon Iver", "Sufjan Stevens"],
  "suno_prompt": "indie folk, acoustic guitar, soft male vocals, 92 bpm, nostalgic warm atmosphere, G major"
}
```

### 5.5 合并模式（与视频路径 2 联动）

当用户在视频文案总结中选择路径 2（音视频合并），且勾选了音乐分析：

- 音乐分析必跑
- **有人声**：转写总结 + 音乐分析同时输出
- **无人声**：仅音乐分析 + 画面描述合并，生成完整的「音画提示词」

### 5.6 三处可单独导出节点（绿色）

1. **原始音频文件**：在下载/读取后立刻可导出
2. **生成的字幕文件**：清洗后可导出 .srt / .txt
3. **最终总结结果**：完成后可导出 .md / .json

## 第十四部分：界面设计要点（给设计师参考）

### 14.1 整体设计原则

1. **任务为核心**：所有功能围绕任务展开，导航始终能回到任务总览
2. **进度可见**：任何耗时操作都有清晰的进度反馈
3. **复刻友好**：核心场景是复刻，提示词、参考帧的展示和复制要极度便捷
4. **少打扰**：异常和转码尽量后台处理，不打断用户

### 14.2 关键页面优先级

**P0（必须做好）：**
- 任务列表 + 任务详情页
- 视频详情页（三轨时间轴）
- 图片详情页（左图右信息）
- 任务勾选面板
- 全流程进度面板

**P1（重要）：**
- 任务级 LLM 对话
- 模型管理设置
- 复刻清单 / 风格报告
- 批量队列面板

**P2（增强体验）：**
- 提示词版本对比
- 生成结果对比原作
- 标签库筛选检索

### 14.3 关键交互细节

- **拖拽上传**：所有可上传位置都支持拖拽
- **一键复制**：所有提示词/标签旁都有复制按钮
- **键盘快捷键**：视频播放、跳转、收藏等核心操作支持快捷键
- **暗色模式**：从设计阶段就考虑

### 14.4 数据展示原则

- 时间戳一律带起点和终点（00:00:05 - 00:00:12）
- JSON 数据默认折叠，点击展开
- 长文本默认显示前几行，点击展开全部
- 缩略图统一规格（视频 16:9，图片保持原比例）

## 第十五部分：已确认事项清单

| 决策点 | 结论 |
|------|------|
| 部署方式 | 本地优先（考虑开源），云端版未来收费 |
| 用户范围 | 个人使用，暂不做团队权限 |
| 任务结构 | 大任务 → 多素材 → 共享数据库 → 任务级 LLM 对话 |
| 内容添加 | 自动识别 + 手动选择 |
| 提示词标签 | 自动归类 + 手动打标双模式 |
| 风格报告触发 | 手动触发 + 自动触发（达阈值）双模式 |
| 对比精度 | 前置让用户选（整体/构图/像素 三档） |
| 工作包分享 | 暂不支持，仅本地导出 |
| 模型配置 | 用户自行添加 API Key + Base URL，不内置 |
| 文件大小限制 | 本地版无限制 |
