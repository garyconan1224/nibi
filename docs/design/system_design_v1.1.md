## 0. 阅读与对齐说明

### 0.1 文档与原型的关系

本文档（spec）与同目录下的 `VidMirror.html`（原型）配对：
- **本文档** 描述「应当如此」的功能与流程
- **原型** 描述「视觉与交互如此」的最终样貌（带可点击 Tweaks，可切换 light/dark、accent、layout、density、lang）
- 后端实现（`nibi/`）尚未完成，下文所有「默认值 / 阈值 / 触发条件」均以 §15 的预设为准

### 0.2 模块命名与代码对齐

原型组件按功能分块，spec 中每个章节对应原型中一个或多个 React 组件：

| Spec 章节 | 原型组件 | 说明 |
|---|---|---|
| §2 任务系统 | `Taskboard` + `TasksDrawer` | 任务列表、任务抽屉 |
| §3 输入层 | `AddMaterialModal` + `QueuePanel` | 添加素材弹窗、批量队列 |
| §4 前置配置 | `AddMaterialModal` 内置区段 | 类型卡片 + 任务 chip |
| §5 视频分支 | `VideoDetail` + `Results` | 播放器 + 三轨时间轴 |
| §6 音频分支 | `AudioDetail` | 波形 + 字幕滚动 |
| §7 图片分支 | `ImageDetail` | 左图右信息 |
| §8 文字分支 | `TextDetail` | 原文 + 分析对照 |
| §9 复刻专项 | `Director` | 标签库 / 收藏夹 / 风格报告 / 对比 / 版本 / 导出 |
| §11 进度可视 | `Processing` | 阶段栈 + 预览 + 日志 |

### 0.3 术语统一

- **任务（Task）** = 一个工作空间，1:N 包含若干素材
- **素材（Material / Item）** = 一个视频/音频/图片/文字 文件
- **阶段（Stage）** = 单个素材在处理流水线中的某一步（见 §11）
- **作业（Job）** = 一次完整的分析执行（绑定模型选择 + 任务勾选）

## 1. 系统架构总览

### 1.1 整体层级

```
应用
├── 任务列表页（首页）
│   └── 新建任务 / 打开已有任务
└── 任务详情页
    ├── 素材列表
    ├── 添加内容（链接 / 本地文件）
    ├── 前置配置面板
    ├── 分析结果视图
    │   ├── 视频结果页（播放器 + 三轨时间轴）
    │   ├── 图片结果页（左图右信息）
    │   ├── 音频结果页（波形图 + 字幕）
    │   └── 文字结果页（原文 + 分析对照）
    ├── 任务级 LLM 对话侧栏
    ├── 复刻清单页
    ├── 风格报告页
    └── 导出工作包入口
```

### 1.2 数据流向

```
用户输入（链接 / 本地文件）
  → 下载 / 读取 / 转码
  → 前置配置（背景信息 + 模型选择 + 任务勾选）
  → 并行分析（视频 / 音频 / 图片 / 文字）
  → 结果存入任务数据库
  → 展示在结果页（可交互、可复制、可收藏）
  → 复刻专项工具（标签库 / 风格报告 / 对比 / 导出）
```

### 1.3 颜色语义（设计时对应使用）

| 颜色 | 语义 | 用途 | 原型 token | 默认 hex |
|------|------|------|-----------|---------|
| 紫色 | 任务系统 / 视频模块 | 任务卡片、视频相关 UI | `--accent-2` | `#B84CFF` |
| 绿色（teal） | 音频模块 / 可导出节点 | 音频相关 UI、绿色导出按钮 | `--accent-green` | `#22D39A` |
| 蓝色 | 图片模块 | 图片相关 UI | `--accent-3` | `#3C77FB` |
| 灰色 | 文字模块 / 系统处理节点 | 文字相关 UI、系统状态 | `--ink-3` | `#7a7a7a` |
| 橙色（amber） | 用户决策节点 | 所有需要用户做选择的弹窗、勾选面板 | `--accent-warm` | `#FFB84C` |
| 红色（coral） | 输入 / 输出层 | 入口按钮、最终输出区 | `--accent` | `#FF4D7E` |
| 深红色 | 复刻专项功能 | 复刻相关工具卡片 | `--accent-deep` | `#C8365A` |

> 在 Tweaks 中切换 accent 时（pink / sunset / lime / mono / purple），上表语义不变，但语义 → hex 的映射会按所选色板等比迁移。详见 §16。

### 1.4 全局状态机（任务 → 素材 → 阶段）

```
[任务级 status]
  draft  → 已建任务但尚未添加素材
  active → 至少一个素材未完成（处理中或队列中）
  done   → 全部素材 done 或 failed（用户已知会）
  archived → 用户归档（不在主列表显示，可在「已归档」中找回）

[素材级 status]
  pending    → 进入队列
  downloading → 仅链接来源
  probing    → 探测格式、时长、字幕轨
  processing → 跑分析（细分阶段见 §11）
  done
  failed     → 终态，可重试
  cancelled  → 用户取消，可恢复

[阶段级 status]（见 §11.1）
  queued · running · done · failed · skipped
```

状态跃迁规则：
- 素材一旦进入 `done`，结果即写入任务数据库，可在结果页查看，**不再因模型变更而自动重跑**（要重跑必须新建作业）
- 任一阶段 `failed` 后，素材整体不立刻置 failed —— 若该阶段是「可跳过」（见 §15.4），则标 `skipped` 后继续；否则才置失败
- 用户在「批量队列」点取消，素材进入 `cancelled`，已完成的中间产物（如已下载文件）保留

## 2. 任务系统

### 2.1 概念定义

**任务 = 一个独立工作空间**

- 一个任务可包含多种内容（视频、音频、图片、文案）
- 同一任务内所有内容共享一个数据库
- 任务数据库支持 LLM 跨内容提问

### 2.2 任务数据结构

```json
{
  "task_id": "uuid",
  "name": "参考素材 - 某创作者风格",
  "created_at": "2024-01-01T00:00:00Z",
  "status": "active | completed | processing",
  "items": [
    {
      "item_id": "uuid",
      "type": "video | audio | image | text",
      "source": "url | local",
      "status": "pending | processing | done | failed",
      "results": {}
    }
  ],
  "database": {
    "prompts": [],
    "subtitles": [],
    "frames": [],
    "music_analysis": [],
    "summaries": [],
    "favorites": []
  }
}
```

### 2.3 任务卡片视觉规范（v1.1 新增）

每张任务卡片至少展示：

```
┌──────────────────────────────────────┐
│ [色条] 任务名称                  ●    │  ← 顶部 4px 色条（按主导素材类型上色）
│ 创建于 3 天前 · 上次活动 2 小时前        │  ← 元信息（mono 字体，ink-3）
├──────────────────────────────────────┤
│ ▢▢▢▢ + 3                              │  ← 缩略图行（最多 4 张 + 计数）
│ 5 视频 · 3 图片 · 1 音频               │  ← 素材组成
├──────────────────────────────────────┤
│ ⟳ 处理中 (2)    ✓ 已完成 (7)          │  ← 状态徽章
│                              ⋯        │  ← 右键菜单触发
└──────────────────────────────────────┘
```

- **悬停态**：卡片上浮 2px + 阴影变化
- **空状态**（任务为空时整页）：插画 + 「新建你的第一个任务」 CTA
- **右键 / 长按**：重命名 / 复制为模板 / 归档 / 删除（删除二次确认）

### 2.4 任务列表页

**页面结构：**
```
顶部导航栏
  └── Logo | 搜索任务 | 新建任务按钮（主操作）

任务卡片网格（每行 2-3 个）
  每张卡片包含：
  - 任务名称
  - 创建时间
  - 素材数量（N 个视频 / M 张图片 / ...）
  - 任务状态标签（处理中 / 已完成）
  - 点击进入任务详情
```

**交互：**
- 点击卡片 → 进入任务详情页
- 点击「新建任务」→ 弹出输入任务名的模态框 → 创建后跳转任务详情页
- 长按 / 右键卡片 → 重命名 / 删除任务

### 2.5 任务详情页

**页面结构：**
```
顶部面包屑：任务列表 > 任务名称

左侧主内容区（约 70% 宽）
  ├── 素材列表（缩略图卡片）
  ├── 添加内容按钮（链接 / 上传）
  └── 当前选中素材的结果视图

右侧侧栏（约 30% 宽）
  ├── 任务级 LLM 对话框（始终可见）
  ├── 复刻清单入口
  └── 导出工作包按钮
```

### 2.6 任务级 LLM 对话

**位置**：任务详情页右侧固定侧栏

**交互逻辑：**
```
用户输入问题
  → 系统将任务内所有分析结果作为上下文注入
  → 调用用户配置的文本大模型
  → 流式输出回答
```

**预设快捷问题（UI 可展示为快捷按钮）：**
- 「这批素材的共同画风提示词有哪些？」
- 「整合所有音乐分析，找共同风格」
- 「哪个素材最值得复刻？」
- 「把所有金句整理成表格」

## 3. 输入层

### 3.1 两种输入入口

#### 入口 A：网络链接

**支持平台与域名识别表（v1.1 新增具体模式）：**

| 平台 | 域名模式（任一命中即匹配） | yt-dlp 策略 |
|------|---------------------|-------------|
| B 站 | `bilibili.com`, `b23.tv` | `bilibili` |
| 小红书 | `xiaohongshu.com`, `xhslink.com` | `generic + cookie` |
| 抖音 | `douyin.com`, `iesdouyin.com`, `v.douyin.com` | `douyin` |
| 快手 | `kuaishou.com`, `kwai.com` | `kuaishou` |
| YouTube | `youtube.com`, `youtu.be`, `m.youtube.com` | `youtube` |
| 推特 | `twitter.com`, `x.com` | `twitter` |
| Instagram | `instagram.com` | `instagram` |
| 网页文章 | 其他 | `readability 提取正文` |

**处理流程（后端逻辑，前端仅展示进度）：**
```
URL 输入
  → URL 格式验证（不合法则提示错误）
  → 平台识别（通过域名匹配）
    ├── 已知平台 → 使用 yt-dlp 对应策略
    └── 未知平台 → 尝试通用网页正文抓取
  → 内容类型检测
    ├── 单一类型（纯视频 / 纯音频 / 纯文章）→ 自动进入
    └── 混合类型（视频+音频 / 图文混合）→ 弹窗让用户选择下载哪些
  → 视频内容时：画质选择（最高 / 1080p / 720p / 仅音频）
  → yt-dlp 执行下载（实时进度条）
  → 下载完成 → 文件存入临时目录 → 可直接导出
```

**下载异常处理：**
- 版权限制 → 「此内容因版权限制无法下载，请尝试其他来源」
- 地区限制 → 「此内容在当前地区不可用」
- 网络超时 → 「下载超时，请检查网络后重试」
- 平台不支持 → 「暂不支持此平台，已尝试通用抓取」

**URL 输入交互细节（v1.1 新增）：**
- 粘贴即检测（debounce 300ms），识别后在输入框左侧显示平台图标 + 名称
- 多条 URL：支持换行或逗号分隔一次性粘贴，逐条验证后进入同一任务的队列
- “快贴”：菜单栏能读取系统剪贴板，检测到 URL 时在右下推荐「使用剪贴板中的链接创建任务」

#### 入口 B：本地文件上传

**支持格式：**
| 类型 | 格式 |
|------|------|
| 视频 | mp4, mov, mkv, avi, wmv, flv, webm |
| 音频 | mp3, wav, aac, flac, m4a, ogg |
| 图片 | jpg, jpeg, png, webp, gif, heic, bmp |
| 文字 | txt, md, pdf, docx, doc |

**处理流程：**
```
文件上传（拖拽 / 点击 / 批量，本地版无大小限制）
  → 文件完整性校验（损坏文件提示重传）
  → 格式识别（通过文件头 magic bytes，非扩展名）
  → 格式判断
    ├── 支持的格式 → 直接进入下一步
    └── 不支持的格式 → 后台自动转码（用户无需操作，显示转码进度条）
  → 读取完成 → 可直接导出
```

### 3.2 可导出原文件（绿色节点）

任何输入内容在下载 / 读取完成后，立刻出现导出按钮：
- 视频 → 导出 .mp4
- 音频 → 导出 .mp3 / .wav
- 图片 → 导出原图
- 文字（网页）→ 导出 .txt（正文）

### 3.3 内容类型路由

```
下载 / 读取完成
  → 识别内容类型
  → 路由到对应分析模块
    ├── 视频 → 视频分支
    ├── 音频 → 音频分支
    ├── 图片 → 图片分支
    └── 文字 → 文字分支
```

### 3.4 批量任务管理

**批量队列面板（当有多个任务时展示）：**
```
队列面板
  ├── 任务项（每行）
  │   ├── 缩略图 / 文件名
  │   ├── 类型标签
  │   ├── 进度条（等待中 / 进行中 / 已完成 / 失败）
  │   ├── 取消按钮（进行中时可用）
  │   └── 重试按钮（失败时可用）
  ├── 整体进度汇总（N 个任务中 M 个已完成）
  └── 暂停全部 / 全部重试 按钮
```

**并行数量上限：**
- 启动时检测电脑性能（CPU 核心数 / 内存 / GPU）
- 自动推荐合理并行数（如：推荐同时处理 3 个任务）
- 用户可在推荐范围内手动调整（滑块或数字输入）

## 4. 前置配置

> 每次添加内容后、开始分析前，弹出前置配置面板

**字段上限与勾选依赖（v1.1 新增）总表：**

| 字段 | 上限 / 默认 | 依赖关系 |
|------|------------|----------|
| 内容类型 | 单选，必填（默认「其他」） | 决定「总结结构」模板 |
| 参与人员 | 最多 12 人，每人名 16 字 | 说话人区分勾选后会取以医配 |
| 主题背景 | ≤ 200 字 | 所有 LLM 调用都会拼接该文本 |
| 专有名词 | 最多 30 个标签，每个 24 字 | 注入 Whisper prompt 提升识别 |
| 分析目的 | 单选 | 决定默认输出格式（复刻参考默认 MJ、竞品分析默认结构化 JSON）|

**勾选项的依赖级联（重要）：**
- 「视频文案总结 - 路径 2（音视频合并）」勾选后，**自动勾选「画面提示词生成」**（复用截帧）且无法取消
- 「说话人音色区分」勾选后，**自动勾选「人声转写」**且无法取消
- 「生成字幕文件」需要「人声转写」，二者同勾同取
- 「多图 / 多文对比」仅在上传 ≥ 2 张同类型素材时可选，否则置灰

### 4.1 面板结构

```
前置配置面板（模态弹窗或侧滑面板）
  ├── 第一区：背景信息（可选）
  ├── 第二区：模型选择
  ├── 第三区：任务勾选
  └── 底部操作：[取消] [开始分析]
```

### 4.2 第一区：背景信息

**作用**：注入到所有后续 AI 调用，提升准确率

| 字段 | 输入类型 | 示例 |
|------|---------|------|
| 内容类型 | 单选下拉 | 课程 / 会议 / 宣传片 / Vlog / 访谈 / 纯音乐 / 其他 |
| 参与人员 | 文本输入（逗号分隔） | 张总, 李总, 产品负责人 |
| 主题背景 | 文本输入 | Q3 战略会议 |
| 专有名词 | 标签输入 | 产品名, 技术术语 |
| 分析目的 | 单选下拉 | 复刻参考 / 竞品分析 / 内容学习 / 其他 |

**注入位置：**
- Whisper 转写：专有名词影响识别准确率
- 说话人匹配：参与人员用于自动匹配声纹标签
- LLM 总结：所有字段作为上下文前置

### 4.3 第二区：模型选择

> **重要**：系统不内置任何模型，全部由用户自行配置

**模型管理（在系统设置中配置，此处只做选择）：**
```
系统设置 → 模型管理
  添加模型：
    - 模型名称（显示名，用户自定义）
    - API Key
    - Base URL
```

## 5. 视频分支

### 5.0 默认参数（v1.1 新增，可在设置中调整）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 按秒截帧间隔 | 2 秒 | 按秒模式默认值 |
| 按秒截帧最大帧数 | 60 张 | 避免超长视频炸费 |
| 相似帧过滤阈值 | pHash 汉明距离 ≤ 6 | 低于该值视为重复帧，丢弃 |
| 镜头识别阈值 | SSIM < 0.55 | 低于该值视为切镜头 |
| AI 镜头默认取帧 | 3 帧（首+中+尾）| 镜头多于 80 个时提示用户限制 |
| 视觉模型重试 | 3 次，指数退避 1s/3s/9s | 超过后该帧标「分析失败」 |
| 提示词默认格式 | Midjourney | 仅影响首次呈现，可一键切换 |


### 5.1 功能 A：画面提示词生成

#### 处理流程（后端）

```
读取视频基本信息（时长 / 分辨率 / 帧率）
  ↓
根据前置勾选的截帧模式分叉：

模式 A：按秒截帧
  → ffmpeg 按用户设置间隔截帧
  → 输出：frame_001.jpg, frame_002.jpg ...
  → 【必须执行】过滤相似帧（对比相邻帧差异，丢弃重复画面）
  → 得到最终帧列表

模式 B：AI 镜头分析
  → 自动识别镜头切换点
  → 展示：「检测到 N 个镜头」（让用户确认后继续）
  → 根据用户选择，每个镜头取帧：
      2 帧模式：取每个镜头的【首帧 + 尾帧】
                适合简单运镜，描述运动起点和终点
      3 帧模式：取每个镜头的【首帧 + 中间帧 + 尾帧】
                适合复杂镜头，中间帧描述过渡状态
  → 【无需过滤相似帧】（按切换点截帧本身就不重复）
  → 直接得到帧列表
  ↓
逐帧送入视觉大模型（用户已配置的视觉模型）
  → 系统 Prompt：要求输出英文描述，维度包含：
      主体（Subject）/ 风格（Style）/ 光线（Lighting）/
      色调（Color tone）/ 构图（Composition）
  → 失败自动重试 3 次，超过则跳过该帧并标注「分析失败」
  ↓
汇总所有帧描述（按时间顺序排列）
  ↓
LLM 精炼为提示词（用户已配置的文本模型）
  → 输入：帧描述汇总 + 用户选择的风格指令
  → 输出格式根据用户选择：
      Midjourney：逗号分隔的英文标签 + 参数（--ar --s 等）
      Stable Diffusion：正向提示词 + 负向提示词
      视频运镜：描述运动轨迹的英文提示词（适合 Runway / Kling）
      JSON：结构化输出（含时间戳 / 帧图 / 描述 / 提示词 / 标签）
  ↓
提示词自动打标签（7 个维度：风格/光线/构图/色调/镜头/人物/场景）
  ↓
结果存入任务数据库
```

#### JSON 输出格式

```json
{

    {
```

### 5.2 功能 B：视频文案总结

#### 前置：字幕检测与处理

```
检测视频是否自带字幕（内嵌字幕轨 / 外挂 .srt .ass）
  ├── 有字幕
  │     → 提取字幕文本
  │     → 原文直接使用（无需清洗）
  │     → ★ 字幕文件可单独导出（绿色导出按钮）
  └── 无字幕
        → 提取音频轨道
        → ★ 音频文件可单独导出
        → Whisper 转写（输出带时间戳的字幕文本）
        → 字幕清洗（去语气词 / 重复句 / 识别错误）
        → ★ 清洗后字幕可单独导出（.srt / .txt）
```

#### 三条总结路径

**路径 1：字幕直接总结**
```
字幕文本
  → 超长文本检测：超出上下文限制则按时间段自动分段
  → 注入背景信息 + 视频类型
  → LLM 生成总结（套用内容类型模板）
  → 输出：标题 / 摘要 / 要点列表 / 金句摘录
```
- 适合：纯语音内容（演讲、课程、播客）
- 速度：最快
- 成本：最低

**路径 2：音视频合并总结（最详细）**
```
字幕文本 + 截帧画面描述（复用路径 A 的截帧模块）
  → 按时间戳对齐合并（字幕文本 ↔ 对应帧描述）
  → 注入背景信息 + 视频类型 + 用户选择的总结深度
  → LLM 生成总结
  → 输出：摘要 / 要点 / 画面引用说明（带截图）
```
- 适合：需要画面引用的场景（宣传片分析、复刻参考）
- 注意：额外消耗截帧 + 视觉模型的时间和费用
- 联动：此路径下，如果勾选了「音乐分析」，音乐分析必跑

**路径 3：视频模型直接分析（最轻量）**
```
整段视频直接送入视频大模型（无需截帧）
  → 模型选择：用户配置的视频模型（Gemini 1.5 Pro / GPT-4o / Qwen-VL 等）
  → 输出：视频内容简单描述
  → 界面标注所用模型名称
```
- 适合：快速预览、不需要详细分析
- 质量：依赖所选模型能力
- 速度：最快（单次 API 调用）

#### 视频类型 × 总结结构对照

| 视频类型 | 总结包含字段 |
|---------|------------|
| 课程 | 知识点列表 / 重点概念 / 学习建议 |
| 宣传片 | 核心卖点 / 目标受众 / 视觉亮点 |
| Vlog | 地点/事件亮点 / 情绪氛围 |
| 访谈 | 受访者核心观点 / 金句摘录 / 话题梳理 |
| 新闻报道 | 事件背景 / 关键信息 / 时间线 |

## 6. 音频分支

### 6.1 前置：音频上传时即可决定是否分析音乐

> 纯音频上传和视频中的音频，都在前置配置勾选阶段决定是否分析音乐，不依赖 VAD 自动判断

### 6.2 VAD 人声检测（决定主流程走向）

**阈值与参数（v1.1 新增）：**
- VAD 人声占比 ≥ **15%** → 走人声处理路径
- 5% ≤ 人声占比 < 15% → 提示「人声较少，是否仅分析音乐？」
- 人声占比 < 5% → 默认当作纯音乐
- 阈值可在 **设置 · 高级** 调整

```
VAD（Voice Activity Detection）人声活动检测
  → 判断人声占比
  ├── 有人声 → 进入「人声处理路径」
  └── 无人声 / 纯音乐
        ├── 用户勾选了「音乐分析」→ 进入「音乐分析路径」
        └── 用户未勾选 → 提示用户「未检测到人声，可返回勾选音乐分析」
```

### 6.3 人声处理路径（详细）

```
步骤 1：Whisper 参数配置（依据前置勾选）
  - 语言：自动检测 / 手动指定
  - 是否开启说话人分离

步骤 2：说话人音色识别（如果勾选）
  → 声纹聚类：自动识别音频中有几个不同说话人
  → 系统提示：「检测到 N 个说话人」
  → 结合背景信息中的参与人员自动尝试匹配
    （例：背景信息填了「张总、李总」→ 自动尝试 Speaker A=张总）
  → 用户确认 / 修正说话人标签（界面：每人展示示例片段 + 输入框）
  → 可跳过（保留 Speaker A / B / C 标签）

步骤 3：Whisper 执行转写
  输出格式：
  [00:00:05][张总] 大家好，今天我们讨论 Q3 战略...
  [00:00:18][李总] 我觉得首先要明确目标客户...
  [00:00:32][Speaker C] 从产品角度看...

步骤 4：文本清洗
  - 去除语气词（嗯、啊、那个）
  - 合并不必要的短句
  - 注入背景信息中的专有名词进行修正

步骤 5：★ 字幕文件可单独导出
  - .srt（带时间戳 + 说话人标注）
  - .txt（纯文本，可选是否保留说话人）

步骤 6：用户选择输出内容（可多选）
  通用：摘要 / 要点 / 金句
  会议场景额外：待办事项 / 决策记录 / 发言人小结

步骤 7：LLM 生成总结
  - 注入：清洗后文本 + 内容类型 + 输出要求 + 背景信息
  - 超长文本自动分段后合并
```

```
  → 读取 EXIF 信息（若存在）
      - 拍摄时间
      - 拍摄地点（GPS 坐标 → 解析为地名）
      - 拍摄设备
  → 展示在结果页的 EXIF 区域
  → EXIF 信息注入联想总结 LLM 调用
```

### 7.2 四个并行任务

> 根据前置勾选，只执行用户选中的任务

#### 任务 1：内容识别描述

```
视觉大模型分析图片
  → 输出维度：
      主体（Subject）：图中核心人物或物品
      场景（Scene）：环境、地点
      色调（Color）：主色、辅色、整体色温
      构图（Composition）：景别（特写/中景/远景）、角度
      风格（Style）：写实 / 插画 / 3D / 水彩等
      细节（Details）：值得注意的特殊元素
  → 输出：结构化描述文本
```

#### 任务 2：OCR 文字提取

```
OCR 引擎识别
  → 支持：中英文混合 / 手写体 / 竖排
  → 文字清洗排版（段落整理 / 标点修正）
  → 输出：提取文字
  → ★ 可单独导出 .txt
```

#### 任务 3：画面提示词生成

```
复用视频截帧模块的视觉模型逻辑（代码共用）
  → 用户选择格式：Midjourney / Stable Diffusion / JSON
  → LLM 精炼
  → 提示词自动打标签入库
```

#### 任务 4：内容联想总结

```
用户选择联想方向（前置勾选中已选）
  ├── 用途推断：图片可能用在哪些场合
  ├── 设计风格分析：设计风格、视觉语言
  ├── 竞品洞察：与市场类似产品对比
  └── 情绪解读：情绪基调、目标用户的感受
  → 注入 EXIF + 背景信息 + 内容描述 → LLM 推导
  → 输出：用途 / 目标受众 / 设计意图 / 竞品洞察
```

### 7.3 多图对比

> 条件：上传多张图片且前置勾选了「多图对比」

```
各图完成四并行任务后
  → 横向对比维度：
      - 风格一致性：是否属于同一系列
      - 差异点：色调、构图、主体的不同之处
      - 共同特征：相似的视觉元素
  → 输出格式选择：
      ○ 表格形式（适合快速对比）
```

```
  → 自动按段落或章节切分
  → 各段独立处理
  → 结果合并（先合并后再做最终总结，非简单拼接）
```

### 8.3 四个并行任务

#### 任务 1：摘要 / 要点 / 金句

```
一次 LLM 调用同时生成：
  - 摘要：一段话（长度：50字 / 100字 / 200字，前置勾选时选择）
  - 要点：编号列表（含原文位置标注）
  - 金句：直接引用（含原文位置标注）
```

#### 任务 2：联想归纳

```
注入背景信息 + 选定方向（前置勾选中已选）
  → LLM 深度推导
  → 输出比摘要更深的洞察层：
      深度解读：文本背后的逻辑
      观点提炼：作者核心主张
      趋势判断：行业/领域趋势
      行动建议：可操作的下一步
```

#### 任务 3：改写 / 润色 / 翻译

```
根据前置勾选：
  改写风格：正式 / 口语 / 简洁 / 丰富
  翻译：输出到目标语言
  → 结果与原文并排对照展示
  → ★ 可导出 .docx（保留对照格式）
```

#### 任务 4：多文对比

```
条件：上传多篇文章且前置勾选了「多文对比」
  → 对比维度：
      观点异同 / 立场倾向 / 信息完整性 / 时间线梳理
  → 输出格式：表格 or 报告
```

## 13. 异常处理汇总

| 异常情况 | 触发条件 | 处理方式 | UI 展示 |
|---------|---------|---------|---------|
| 链接下载失败 | 版权/地区/网络 | 记录错误码 | 红色提示 + 具体原因 + 重试按钮 |
| 平台不支持 | 域名不在支持列表 | 尝试通用抓取 | 黄色提示「正在尝试通用抓取」 |
| 文件损坏 | magic bytes 校验失败 | 拒绝处理 | 红色提示「文件损坏，请重新上传」 |
| 格式不支持 | 格式识别后无法处理 | 后台自动转码 | 灰色提示「正在转换格式...」 |
| 无人声 | VAD 检测人声占比<阈值 | 提示用户 | 「未检测到人声，是否分析音乐？」 |
| 视觉模型 API 失败 | HTTP 错误 / 超时 | 自动重试 3 次 | 进度条暂停 + 「重试中 1/3」 |
| 超过重试次数 | 第 3 次仍失败 | 跳过该帧并标注 | 该帧缩略图显示「分析失败」标记 |
| 文本超出上下文 | token 数超限 | 自动分段处理 | 灰色提示「文本较长，自动分段处理」 |
| 批量任务单个失败 | 任一子任务异常 | 标红，不影响其他 | 红色状态 + 单独重试按钮 |
| 模型未配置 | 未添加任何模型 | 拦截执行 | 弹出「请先在设置中添加模型」引导 |

## 15. 默认预设（Presets）

> **说明**：后端 nibi 尚未完成。本章集中列出所有阈值、默认值与可调范围，作为前后端契约。原型中以该表为准呈现「默认体验」。

### 15.1 模型预设模板

首次进入设置页时，提供三个模板一键导入（仅写入「名称 / Base URL / 模型类型」，API Key 留空要求用户填）：

| 模板 | 包含项 | Base URL |
|------|--------|----------|
| **OpenAI 兼容** | `gpt-4o`（视觉）、`gpt-4o-mini`（文本） | `https://api.openai.com/v1` |
| **Anthropic** | `claude-sonnet-4`（视觉+文本） | `https://api.anthropic.com/v1` |
| **本地 Ollama** | `qwen2.5vl:7b`（视觉）、`qwen2.5:7b`（文本） | `http://localhost:11434/v1` |

用户未配置任何模型时，「开始分析」被拦截，弹出「请先在设置中添加模型」引导（引用 §4.3）。

### 15.2 任务预设（快捷模板）

「添加素材」弹窗顶部提供预设快捷，一键勾选一组任务：

| 预设名 | 场景 | 默认勾选 |
|--------|------|-----------|
| **复刻参考** | 拆解别人的跳转 | 画面提示词（AI 镜头/3 帧） + 文案总结（路径1） + 音乐分析 |
| **字幕优先** | 仅要记录讲话内容 | 人声转写 + 字幕导出（不跑截帧） |
| **快速预览** | 丰富素材筛选 | 仅「视频文案总结（路径3、视频模型）」 |
| **会议记录** | 钉钉/腾讯会议导入 | 人声转写 + 说话人区分 + 要点/待办/决策输出 |
| **自定义** | 隐藏预设，用户手动勾选 | — |

选中预设后仍可手动勾除，但依赖级联仍生效（§4.4 机制）。

### 15.3 性能预设（并行数量）

首次启动探测 CPU 核心 / 内存 / GPU，按下表推荐：

| 机型 | 推荐并行 | 最大允许 |
|------|---------|----------|
| 轻量（4 核 / 8GB / 集显） | 1 | 2 |
| 中型（8 核 / 16GB / 4GB 显存） | 2 | 4 |
| 重度（16+ 核 / 32GB+ / 8GB+ 显存） | 4 | 8 |

用户可在「设置 · 性能」中调整，超出最大允许时接受提示但不禁止。

### 15.4 可跳过阶段与限制（重要）

表示某阶段失败不终结整个作业、该阶段标 `skipped` 后继续：

| 阶段 | 可跳过？ | 跳过后产物 |
|------|--------|-----------|
| `download` | 否 | — |
| `probe` | 否 | — |
| `frames` | 是 | 总结路径退化为路径 1 |
| `asr` | 是 | 字幕为空，总结跳过文本分析 |
| `vlm` | 部分可跳 | 单帧失败在该帧标「分析失败」，总体不失败 |
| `sum` | 否 | —（总结是交付的价值所在） |
| `store` | 否 | — |

### 15.5 存储与清理策略

- 中间产物（截帧、提取音频、转码后视频）默认保留 **14 天**，任务归档后保留 **3 天**
- 原始下载文件（mp4/mp3/原图）默认 **不自动清理**，仅在用户手动删除任务时一同删
- 提示词与总结等文本产物 **永久保留**，仅随任务删除

## 16. 设计令牌（Design Tokens）

> 从 VidMirror.html 原型反向沉淀，后端生成报告 / 邮件等 UI 产物时可复用。

### 16.1 色板（默认 pink accent）

```css
--accent:        #FF4D7E;
--accent-2:      #B84CFF;
--accent-3:      #3C77FB;
--accent-warm:   #FFB84C;
--accent-green:  #22D39A;
--accent-deep:   #C8365A;

--ink:    #1A1A1A;
--ink-2:  #4A4A4A;
--ink-3:  #7A7A7A;
--ink-4:  #B0B0B0;
--line:        rgba(0,0,0,0.08);
--line-strong: rgba(0,0,0,0.16);
--bg:        #FAFAF7;
--bg-elev:   #FFFFFF;
--bg-sunken: #F2F2EE;
```

暗色主题（`data-theme="dark"`）反转 ink / bg 色阶，保持 accent 不变。

### 16.2 字体

- **display**：`Instrument Serif`（italic 可选）
- **sans**：系统无衬线字栈（Pingfang SC / Helvetica Neue / Arial）
- **mono**：`JetBrains Mono`，letter-spacing 0.08em 在徽章场景使用

### 16.3 圆角与阴影

```css
--radius:    14px;
--radius-lg: 20px;
--radius-sm: 8px;
--radius-pill: 999px;
--shadow-sm: 0 1px 2px rgba(0,0,0,0.04);
--shadow-md: 0 6px 18px rgba(0,0,0,0.06), 0 2px 4px rgba(0,0,0,0.04);
--shadow-lg: 0 20px 50px rgba(0,0,0,0.10), 0 6px 14px rgba(0,0,0,0.06);
```

### 16.4 密度× Tweaks

Tweaks 中 `density` 三档：
- `compact`（当前默认）：行高 ×0.92，卡片 padding ×0.85
- `cozy`：基准 ×1
- `roomy`：×1.15

### 16.5 动作

- 默认 easing：`cubic-bezier(0.4, 0, 0.2, 1)`
- 状态过渡 120–220ms；面板出入 280ms；进度条插值 400ms
- 避免 > 400ms 装饰动画；不用弹跳 / overshoot
