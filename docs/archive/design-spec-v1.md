## 1. 系统架构总览

### 1.1 整体层级

```
应用
├── 任务列表页（首页）
│   └── 新建任务 / 打开已有任务
└── 任务详情页
    ├── 素材列表
    ├── 添加内容（链接 / 本地文件）
    ├── 前置配置面板
    ├── 分析结果视图
    │   ├── 视频结果页（播放器 + 三轨时间轴）
    │   ├── 图片结果页（左图右信息）
    │   ├── 音频结果页（波形图 + 字幕）
    │   └── 文字结果页（原文 + 分析对照）
    ├── 任务级 LLM 对话侧栏
    ├── 复刻清单页
    ├── 风格报告页
    └── 导出工作包入口
```

### 1.2 数据流向

```
用户输入（链接 / 本地文件）
  → 下载 / 读取 / 转码
  → 前置配置（背景信息 + 模型选择 + 任务勾选）
  → 并行分析（视频 / 音频 / 图片 / 文字）
  → 结果存入任务数据库
  → 展示在结果页（可交互、可复制、可收藏）
  → 复刻专项工具（标签库 / 风格报告 / 对比 / 导出）
```

### 1.3 颜色语义（设计时对应使用）

| 颜色 | 语义 | 用途 |
|------|------|------|
| 紫色 | 任务系统 / 视频模块 | 任务卡片、视频相关 UI |
| 绿色（teal） | 音频模块 / 可导出节点 | 音频相关 UI、绿色导出按钮 |
| 蓝色 | 图片模块 | 图片相关 UI |
| 灰色 | 文字模块 / 系统处理节点 | 文字相关 UI、系统状态 |
| 橙色（amber） | 用户决策节点 | 所有需要用户做选择的弹窗、勾选面板 |
| 红色（coral） | 输入 / 输出层 | 入口按钮、最终输出区 |
| 深红色 | 复刻专项功能 | 复刻相关工具卡片 |

## 2. 任务系统

### 2.1 概念定义

**任务 = 一个独立工作空间**

- 一个任务可包含多种内容（视频、音频、图片、文案）
- 同一任务内所有内容共享一个数据库
- 任务数据库支持 LLM 跨内容提问

### 2.2 任务数据结构

```json
{
  "task_id": "uuid",
  "name": "参考素材 - 某创作者风格",
  "created_at": "2024-01-01T00:00:00Z",
  "status": "active | completed | processing",
  "items": [
    {
      "item_id": "uuid",
      "type": "video | audio | image | text",
      "source": "url | local",
      "status": "pending | processing | done | failed",
      "results": {}
    }
  ],
  "database": {
    "prompts": [],
    "subtitles": [],
    "frames": [],
    "music_analysis": [],
    "summaries": [],
    "favorites": []
  }
}
```

### 2.3 任务列表页

**页面结构：**
```
顶部导航栏
  └── Logo | 搜索任务 | 新建任务按钮（主操作）

任务卡片网格（每行 2-3 个）
  每张卡片包含：
  - 任务名称
  - 创建时间
  - 素材数量（N 个视频 / M 张图片 / ...）
  - 任务状态标签（处理中 / 已完成）
  - 点击进入任务详情
```

**交互：**
- 点击卡片 → 进入任务详情页
- 点击「新建任务」→ 弹出输入任务名的模态框 → 创建后跳转任务详情页
- 长按 / 右键卡片 → 重命名 / 删除任务

### 2.4 任务详情页

**页面结构：**
```
顶部面包屑：任务列表 > 任务名称

左侧主内容区（约 70% 宽）
  ├── 素材列表（缩略图卡片）
  ├── 添加内容按钮（链接 / 上传）
  └── 当前选中素材的结果视图

右侧侧栏（约 30% 宽）
  ├── 任务级 LLM 对话框（始终可见）
  ├── 复刻清单入口
  └── 导出工作包按钮
```

### 2.5 任务级 LLM 对话

**位置**：任务详情页右侧固定侧栏

**交互逻辑：**
```
用户输入问题
  → 系统将任务内所有分析结果作为上下文注入
  → 调用用户配置的文本大模型
  → 流式输出回答
```

**预设快捷问题（UI 可展示为快捷按钮）：**
- 「这批素材的共同画风提示词有哪些？」
- 「整合所有音乐分析，找共同风格」
- 「哪个素材最值得复刻？」
- 「把所有金句整理成表格」

## 3. 输入层

### 3.1 两种输入入口

#### 入口 A：网络链接

**支持平台：** B 站 / 小红书 / 抖音 / 快手 / YouTube / 网页文章

**处理流程（后端逻辑，前端仅展示进度）：**
```
URL 输入
  → URL 格式验证（不合法则提示错误）
  → 平台识别（通过域名匹配）
    ├── 已知平台 → 使用 yt-dlp 对应策略
    └── 未知平台 → 尝试通用网页正文抓取
  → 内容类型检测
    ├── 单一类型（纯视频 / 纯音频 / 纯文章）→ 自动进入
    └── 混合类型（视频+音频 / 图文混合）→ 弹窗让用户选择下载哪些
  → 视频内容时：画质选择（最高 / 1080p / 720p / 仅音频）
  → yt-dlp 执行下载（实时进度条）
  → 下载完成 → 文件存入临时目录 → 可直接导出
```

**下载异常处理：**
- 版权限制 → 「此内容因版权限制无法下载，请尝试其他来源」
- 地区限制 → 「此内容在当前地区不可用」
- 网络超时 → 「下载超时，请检查网络后重试」
- 平台不支持 → 「暂不支持此平台，已尝试通用抓取」

#### 入口 B：本地文件上传

**支持格式：**
| 类型 | 格式 |
|------|------|
| 视频 | mp4, mov, mkv, avi, wmv, flv, webm |
| 音频 | mp3, wav, aac, flac, m4a, ogg |
| 图片 | jpg, jpeg, png, webp, gif, heic, bmp |
| 文字 | txt, md, pdf, docx, doc |

**处理流程：**
```
文件上传（拖拽 / 点击 / 批量，本地版无大小限制）
  → 文件完整性校验（损坏文件提示重传）
  → 格式识别（通过文件头 magic bytes，非扩展名）
  → 格式判断
    ├── 支持的格式 → 直接进入下一步
    └── 不支持的格式 → 后台自动转码（用户无需操作，显示转码进度条）
  → 读取完成 → 可直接导出
```

### 3.2 可导出原文件（绿色节点）

任何输入内容在下载 / 读取完成后，立刻出现导出按钮：
- 视频 → 导出 .mp4
- 音频 → 导出 .mp3 / .wav
- 图片 → 导出原图
- 文字（网页）→ 导出 .txt（正文）

### 3.3 内容类型路由

```
下载 / 读取完成
  → 识别内容类型
  → 路由到对应分析模块
    ├── 视频 → 视频分支
    ├── 音频 → 音频分支
    ├── 图片 → 图片分支
    └── 文字 → 文字分支
```

### 3.4 批量任务管理

**批量队列面板（当有多个任务时展示）：**
```
队列面板
  ├── 任务项（每行）
  │   ├── 缩略图 / 文件名
  │   ├── 类型标签
  │   ├── 进度条（等待中 / 进行中 / 已完成 / 失败）
  │   ├── 取消按钮（进行中时可用）
  │   └── 重试按钮（失败时可用）
  ├── 整体进度汇总（N 个任务中 M 个已完成）
  └── 暂停全部 / 全部重试 按钮
```

**并行数量上限：**
- 启动时检测电脑性能（CPU 核心数 / 内存 / GPU）
- 自动推荐合理并行数（如：推荐同时处理 3 个任务）
- 用户可在推荐范围内手动调整（滑块或数字输入）

## 4. 前置配置

> 每次添加内容后、开始分析前，弹出前置配置面板

### 4.1 面板结构

```
前置配置面板（模态弹窗或侧滑面板）
  ├── 第一区：背景信息（可选）
  ├── 第二区：模型选择
  ├── 第三区：任务勾选
  └── 底部操作：[取消] [开始分析]
```

### 4.2 第一区：背景信息

**作用**：注入到所有后续 AI 调用，提升准确率

| 字段 | 输入类型 | 示例 |
|------|---------|------|
| 内容类型 | 单选下拉 | 课程 / 会议 / 宣传片 / Vlog / 访谈 / 纯音乐 / 其他 |
| 参与人员 | 文本输入（逗号分隔） | 张总, 李总, 产品负责人 |
| 主题背景 | 文本输入 | Q3 战略会议 |
| 专有名词 | 标签输入 | 产品名, 技术术语 |
| 分析目的 | 单选下拉 | 复刻参考 / 竞品分析 / 内容学习 / 其他 |

**注入位置：**
- Whisper 转写：专有名词影响识别准确率
- 说话人匹配：参与人员用于自动匹配声纹标签
- LLM 总结：所有字段作为上下文前置

### 4.3 第二区：模型选择

> **重要**：系统不内置任何模型，全部由用户自行配置

**模型管理（在系统设置中配置，此处只做选择）：**
```
系统设置 → 模型管理
  添加模型：
    - 模型名称（显示名，用户自定义）
    - API Key
    - Base URL
    - 模型类型：文本模型 / 视觉模型 / 视频模型

支持格式：
  - OpenAI API 兼容格式（GPT-4V, Claude, Qwen 等）
  - Anthropic 原生格式
  - 本地 Ollama
```

**前置配置中的模型选择区域：**
```
模型选择
  ├── 视觉大模型（用于截帧分析 / 图片分析）
  │   └── 下拉选择已配置的视觉模型
  ├── 文本大模型（用于总结 / 归纳 / 对话）
  │   └── 下拉选择已配置的文本模型
  └── 视频大模型（用于视频直接分析路径3，仅勾选时显示）
      └── 下拉选择已配置的视频模型

未配置模型时：显示「前往设置添加模型」引导
```

### 4.4 第三区：任务勾选

> **核心交互**：用户选择本次要执行的分析项，未勾选的完全跳过

**根据内容类型显示不同勾选项：**

#### 视频内容勾选项：
```
☐ 画面提示词生成
    └── 子选项（勾选后展开）
        ├── 截帧模式：○ 按秒截帧  ○ AI 镜头分析
        ├── 按秒模式：每 [__] 秒截 1 张，最大 [__] 张
        └── 镜头模式：每个镜头取 ○ 2 帧（首+尾）  ○ 3 帧（首+中+尾）

☐ 视频文案总结
    └── 子选项（勾选后展开）
        ├── 总结路径：○ 字幕直接总结  ○ 音视频合并（最详细）  ○ 视频模型直接分析
        └── 总结深度（路径2可选）：○ 简洁  ○ 详细  ○ 带画面引用

☐ 音乐分析（分析视频中的背景音乐）
    └── 子选项（勾选后展开）
        └── ☐ 同时生成 Suno / Udio 格式提示词

☐ 字幕导出（转写完成后导出 .srt 文件）
```

#### 音频内容勾选项：
```
☐ 人声转写 + 内容总结
    └── 子选项（勾选后展开）
        ├── 语言：○ 自动检测  ○ 中文  ○ 英文  ○ 其他
        └── ☐ 开启说话人分离（会议 / 采访推荐）

☐ 说话人音色区分
    └── 需配合「人声转写」使用

☐ 生成字幕文件（.srt / .txt）

☐ 音乐分析（分析音频中的音乐特征）
    └── 子选项
        └── ☐ 生成 Suno / Udio 格式提示词
```

#### 图片内容勾选项：
```
☐ 内容识别描述（主体 / 场景 / 色调 / 构图 / 风格）
☐ OCR 文字提取
☐ 画面提示词生成
    └── 子选项：风格 ○ Midjourney  ○ Stable Diffusion  ○ JSON 格式
☐ 内容联想总结
    └── 子选项：方向 ○ 用途推断  ○ 设计分析  ○ 竞品洞察  ○ 情绪解读
☐ 多图对比分析（上传多张时可选）
```

#### 文字内容勾选项：
```
☐ 摘要 / 要点 / 金句
    └── 子选项：摘要长度 ○ 50字  ○ 100字  ○ 200字
☐ 联想归纳
    └── 子选项：方向 ○ 深度解读  ○ 观点提炼  ○ 趋势判断  ○ 行动建议
☐ 改写 / 润色
    └── 子选项：风格 ○ 正式  ○ 口语  ○ 简洁  ○ 丰富
☐ 翻译
    └── 子选项：目标语言（文本输入）
☐ 多文对比（上传多篇时可选）
```

## 5. 视频分支

### 5.1 功能 A：画面提示词生成

#### 处理流程（后端）

```
读取视频基本信息（时长 / 分辨率 / 帧率）
  ↓
根据前置勾选的截帧模式分叉：

模式 A：按秒截帧
  → ffmpeg 按用户设置间隔截帧
  → 输出：frame_001.jpg, frame_002.jpg ...
  → 【必须执行】过滤相似帧（对比相邻帧差异，丢弃重复画面）
  → 得到最终帧列表

模式 B：AI 镜头分析
  → 自动识别镜头切换点
  → 展示：「检测到 N 个镜头」（让用户确认后继续）
  → 根据用户选择，每个镜头取帧：
      2 帧模式：取每个镜头的【首帧 + 尾帧】
                适合简单运镜，描述运动起点和终点
      3 帧模式：取每个镜头的【首帧 + 中间帧 + 尾帧】
                适合复杂镜头，中间帧描述过渡状态
  → 【无需过滤相似帧】（按切换点截帧本身就不重复）
  → 直接得到帧列表
  ↓
逐帧送入视觉大模型（用户已配置的视觉模型）
  → 系统 Prompt：要求输出英文描述，维度包含：
      主体（Subject）/ 风格（Style）/ 光线（Lighting）/
      色调（Color tone）/ 构图（Composition）
  → 失败自动重试 3 次，超过则跳过该帧并标注「分析失败」
  ↓
汇总所有帧描述（按时间顺序排列）
  ↓
LLM 精炼为提示词（用户已配置的文本模型）
  → 输入：帧描述汇总 + 用户选择的风格指令
  → 输出格式根据用户选择：
      Midjourney：逗号分隔的英文标签 + 参数（--ar --s 等）
      Stable Diffusion：正向提示词 + 负向提示词
      视频运镜：描述运动轨迹的英文提示词（适合 Runway / Kling）
      JSON：结构化输出（含时间戳 / 帧图 / 描述 / 提示词 / 标签）
  ↓
提示词自动打标签（7 个维度：风格/光线/构图/色调/镜头/人物/场景）
  ↓
结果存入任务数据库
```

#### JSON 输出格式

```json
{
  "video_id": "uuid",
  "overall_style": "cinematic, warm tones, portrait photography",
  "frames": [
    {
      "timestamp": "00:00:05",
      "timestamp_end": "00:00:08",
      "shot_type": "close_up",
      "thumbnail": "frame_001.jpg",
      "description": "A woman in a red dress standing near a window, soft backlight, warm golden tones",
      "prompt_mj": "woman in red dress, near window, soft backlight, golden hour, cinematic, warm tones --ar 16:9 --s 750",
      "prompt_sd": {
        "positive": "woman in red dress, window light, cinematic lighting, warm tones, soft bokeh",
        "negative": "blurry, low quality, distorted"
      },
      "prompt_video": "camera holds steady, subject stands near window, soft golden backlight, warm intimate atmosphere",
      "tags": {
        "style": ["cinematic", "portrait"],
        "lighting": ["backlight", "golden hour"],
        "composition": ["close-up", "rule of thirds"],
        "color": ["warm tones", "red"],
        "lens": ["soft bokeh"],
        "subject": ["woman", "portrait"],
        "scene": ["indoor", "window"]
      }
    }
  ]
}
```

### 5.2 功能 B：视频文案总结

#### 前置：字幕检测与处理

```
检测视频是否自带字幕（内嵌字幕轨 / 外挂 .srt .ass）
  ├── 有字幕
  │     → 提取字幕文本
  │     → 原文直接使用（无需清洗）
  │     → ★ 字幕文件可单独导出（绿色导出按钮）
  └── 无字幕
        → 提取音频轨道
        → ★ 音频文件可单独导出
        → Whisper 转写（输出带时间戳的字幕文本）
        → 字幕清洗（去语气词 / 重复句 / 识别错误）
        → ★ 清洗后字幕可单独导出（.srt / .txt）
```

#### 三条总结路径

**路径 1：字幕直接总结**
```
字幕文本
  → 超长文本检测：超出上下文限制则按时间段自动分段
  → 注入背景信息 + 视频类型
  → LLM 生成总结（套用内容类型模板）
  → 输出：标题 / 摘要 / 要点列表 / 金句摘录
```
- 适合：纯语音内容（演讲、课程、播客）
- 速度：最快
- 成本：最低

**路径 2：音视频合并总结（最详细）**
```
字幕文本 + 截帧画面描述（复用路径 A 的截帧模块）
  → 按时间戳对齐合并（字幕文本 ↔ 对应帧描述）
  → 注入背景信息 + 视频类型 + 用户选择的总结深度
  → LLM 生成总结
  → 输出：摘要 / 要点 / 画面引用说明（带截图）
```
- 适合：需要画面引用的场景（宣传片分析、复刻参考）
- 注意：额外消耗截帧 + 视觉模型的时间和费用
- 联动：此路径下，如果勾选了「音乐分析」，音乐分析必跑

**路径 3：视频模型直接分析（最轻量）**
```
整段视频直接送入视频大模型（无需截帧）
  → 模型选择：用户配置的视频模型（Gemini 1.5 Pro / GPT-4o / Qwen-VL 等）
  → 输出：视频内容简单描述
  → 界面标注所用模型名称
```
- 适合：快速预览、不需要详细分析
- 质量：依赖所选模型能力
- 速度：最快（单次 API 调用）

#### 视频类型 × 总结结构对照

| 视频类型 | 总结包含字段 |
|---------|------------|
| 课程 | 知识点列表 / 重点概念 / 学习建议 |
| 宣传片 | 核心卖点 / 目标受众 / 视觉亮点 |
| Vlog | 地点/事件亮点 / 情绪氛围 |

| 新闻报道 | 事件背景 / 关键信息 / 时间线 |

## 6. 音频分支

### 6.1 前置：音频上传时即可决定是否分析音乐

> 纯音频上传和视频中的音频，都在前置配置勾选阶段决定是否分析音乐，不依赖 VAD 自动判断

### 6.2 VAD 人声检测（决定主流程走向）

```
VAD（Voice Activity Detection）人声活动检测
  → 判断人声占比
  ├── 有人声 → 进入「人声处理路径」
  └── 无人声 / 纯音乐
        ├── 用户勾选了「音乐分析」→ 进入「音乐分析路径」
        └── 用户未勾选 → 提示用户「未检测到人声，可返回勾选音乐分析」
```

### 6.3 人声处理路径（详细）

```
步骤 1：Whisper 参数配置（依据前置勾选）
  - 语言：自动检测 / 手动指定
  - 是否开启说话人分离

步骤 2：说话人音色识别（如果勾选）
  → 声纹聚类：自动识别音频中有几个不同说话人
  → 系统提示：「检测到 N 个说话人」
  → 结合背景信息中的参与人员自动尝试匹配
    （例：背景信息填了「张总、李总」→ 自动尝试 Speaker A=张总）
  → 用户确认 / 修正说话人标签（界面：每人展示示例片段 + 输入框）
  → 可跳过（保留 Speaker A / B / C 标签）

步骤 3：Whisper 执行转写
  输出格式：
  [00:00:05][张总] 大家好，今天我们讨论 Q3 战略...
  [00:00:18][李总] 我觉得首先要明确目标客户...
  [00:00:32][Speaker C] 从产品角度看...

步骤 4：文本清洗
  - 去除语气词（嗯、啊、那个）
  - 合并不必要的短句
  - 注入背景信息中的专有名词进行修正

步骤 5：★ 字幕文件可单独导出
  - .srt（带时间戳 + 说话人标注）
  - .txt（纯文本，可选是否保留说话人）

步骤 6：用户选择输出内容（可多选）
  通用：摘要 / 要点 / 金句
  会议场景额外：待办事项 / 决策记录 / 发言人小结

步骤 7：LLM 生成总结
  - 注入：清洗后文本 + 内容类型 + 输出要求 + 背景信息
  - 超长文本自动分段后合并
```

**内容类型 × 输出字段：**

| 内容类型 | 通用字段 | 特有字段 |
|---------|---------|---------|
| 播客 | 摘要 / 要点 / 金句 | 话题亮点 / 嘉宾观点 |
| 会议录音 | 摘要 / 要点 | 待办事项 / 决策记录 / 发言人小结 |
| 课程讲解 | 摘要 / 要点 | 知识点 / 重点概念 |
| 演讲 | 摘要 / 要点 / 金句 | 核心论点 |
| 采访 | 摘要 / 要点 / 金句 | 受访者观点 / 问答摘要 |

### 6.4 音乐分析路径（详细）

> **触发条件**：用户在前置勾选了「音乐分析」（不依赖是否有人声）

```
步骤 1：音乐片段切分
  - 按时间段切分音频
  - 每段标注起止时间戳区间
  - 示例：00:00:00-00:01:30 第一段 / 00:01:30-00:03:15 第二段

步骤 2：每段音乐特征分析
  分析维度：
  - 风格（Style）：流行 / 爵士 / 电子 / 古典 / 民谣等
  - 情绪（Mood）：欢快 / 悲伤 / 激昂 / 舒缓等
  - BPM：每分钟节拍数
  - 主要乐器（Instruments）：钢琴 / 吉他 / 合成器等
  - 调性（Key）：C 大调 / A 小调等
  - 整体氛围（Atmosphere）：温暖 / 冷峻 / 神秘等

步骤 3：LLM 生成音乐提示词
  输出：
  - 相似曲风参考方向
  - 适合使用场景（广告 / Vlog 配乐 / 电影配乐等）
  - AI 音乐生成提示词（Suno / Udio 格式）
```

**音乐分析 JSON 输出格式：**

```json
{
  "segments": [
    {
      "timestamp_start": "00:00:00",
      "timestamp_end": "00:01:30",
      "style": "indie folk",
      "mood": "nostalgic, warm",
      "bpm": 92,
      "instruments": ["acoustic guitar", "soft vocals", "light percussion"],
      "key": "G major",
      "atmosphere": "intimate, contemplative"
    }
  ],
  "overall_style": "modern indie folk with cinematic touches",
  "similar_references": ["Bon Iver", "Sufjan Stevens"],
  "use_cases": ["vlog background", "documentary score", "emotional scenes"],
  "suno_prompt": "indie folk, acoustic guitar, soft male vocals, 92 bpm, nostalgic warm atmosphere, G major",
  "udio_prompt": "folk, acoustic, intimate, 92bpm, warm, nostalgic"
}
```

### 6.5 音视频合并联动（重要）

当视频文案总结选择路径 2（音视频合并）且勾选了「音乐分析」时：

- 音乐分析**必须执行**，无论视频中有无人声
- **有人声**：转写总结 + 音乐分析两路并行，结果同时展示
- **无人声**：音乐分析结果 + 视频画面描述合并，生成「完整音画提示词」

```
      ○ 精细像素级对比（细节/纹理/色彩精确度，约 60 秒）
  → 系统自动配对任务内对应参考帧
  → 视觉大模型分析差异
  → 输出对比报告：
      相似度评分：XX / 100
      差距分析：（按所选精度维度展开）
      提示词优化建议：（具体的词语修改建议）
```

**UI 布局：**
```

┌──────────────┬──────────────┐
│  参考原作     │  你的生成结果  │
└──────────────┴──────────────┘

     相似度：72 / 100
     差距：色调偏冷，主体比例偏小...

```

## 13. 异常处理汇总

| 异常情况 | 触发条件 | 处理方式 | UI 展示 |
|---------|---------|---------|---------|
| 链接下载失败 | 版权/地区/网络 | 记录错误码 | 红色提示 + 具体原因 + 重试按钮 |
| 平台不支持 | 域名不在支持列表 | 尝试通用抓取 | 黄色提示「正在尝试通用抓取」 |
| 文件损坏 | magic bytes 校验失败 | 拒绝处理 | 红色提示「文件损坏，请重新上传」 |
| 格式不支持 | 格式识别后无法处理 | 后台自动转码 | 灰色提示「正在转换格式...」 |
| 无人声 | VAD 检测人声占比<阈值 | 提示用户 | 「未检测到人声，是否分析音乐？」 |
| 视觉模型 API 失败 | HTTP 错误 / 超时 | 自动重试 3 次 | 进度条暂停 + 「重试中 1/3」 |
| 超过重试次数 | 第 3 次仍失败 | 跳过该帧并标注 | 该帧缩略图显示「分析失败」标记 |
| 文本超出上下文 | token 数超限 | 自动分段处理 | 灰色提示「文本较长，自动分段处理」 |
| 批量任务单个失败 | 任一子任务异常 | 标红，不影响其他 | 红色状态 + 单独重试按钮 |
| 模型未配置 | 未添加任何模型 | 拦截执行 | 弹出「请先在设置中添加模型」引导 |

## 附录：关键设计原则

1. **任务优先**：所有操作都在任务上下文内，导航始终能回到任务详情
2. **进度可见**：任何耗时超过 2 秒的操作都有进度反馈
3. **复刻友好**：提示词、参考帧的展示和复制是最高频操作，要最便捷
4. **不打扰**：格式转码、异常重试尽量后台静默处理
5. **灵活导出**：任何中间产物（原文件、字幕、提示词）都可单独导出，不强制跑完全流程
6. **暗色模式**：从设计阶段就同时设计亮色和暗色方案
7. **本地感**：设计风格应体现专业本地工具感，非 SaaS 风格
